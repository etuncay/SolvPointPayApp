# 4. Para Gönder

> Ortak kurallar için bkz. `0.genel-standartlar.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Para Gönder |
| Dosya | `4.para-gonder.md` |
| Ekran tipi | Menü grubu / akış başlangıcı |
| Alt ekranlar | 4.1 Kayıtlı Kişilerim, 4.2 Yurt İçine, 4.3 Yurt Dışına |

## 2. Amaç ve Kapsam

Müşterinin self-servis kanaldan para göndermesini sağlayan menüdür. Üç alt akış: kayıtlı kişiler üzerinden hızlı gönderim, yurt içine ve yurt dışına gönderim.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | Para Gönder |
|---|---|
| Bireysel Müşteri | ✓ |
| Tüzel Müşteri Yetkili Kişisi | ✓ (işlem yetkisi dahilinde) |

## 4. Menü ve Navigasyon

Sol menüde "Para Gönder"; altında 4.1 Kayıtlı Kişilerim, 4.2 Yurt İçine, 4.3 Yurt Dışına. İşlem özeti İşlem Onay (1.1) ekranında.

## 5. Ekran Yapısı ve Bileşenler

Menü grubudur; gönderim formları alt ekranlardadır. Gönderim **yalnızca CustomerPersistent cüzdandan** yapılabilir.

## 6. Alan Listesi

Alanlar alt ekran dosyalarında (4.1–4.3) detaylandırılmıştır.

## 7. Validasyon Kuralları

Alt akış bazlı; bkz. 4.1–4.3.

## 8. İş Kuralları ve Akışlar

- Para gönderme **self-servis kanaldan** yapılır; kaynak cüzdan CustomerPersistent olmalıdır.
- UI'da alt akışlar ayrışsa da sistemde tek `transaction` kaydı; farklar `transaction_type` ve ilgili alanlar (IBAN, hedef ülke/para birimi) ile belirlenir.
- Para gönderme öncesi gerekli sanction/limit/fraud kontrolleri uygulanır (bkz. ortak kurallar).
- İşlem özeti İşlem Onay (1.1) ekranında OTP ile onaylanır.

## 9. Statü Modeli

`transaction_status` — bkz. ortak para transferi statü modeli.

## 10. Veri Modeli

`transaction`, `wallet`, `customer_saved_recipient` (kayıtlı kişiler — bkz. 4.1).

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `POST /customer/transfers` | POST | Transfer başlat (alt akış tipi ile; idempotent) |

## 12. Entegrasyonlar

Sanction screening, FX (yurt dışı), banka transfer.

## 13. Yetkilendirme ve Güvenlik

- Yalnızca giriş yapan müşteri / işlem yetkili tüzel yetkili kişi.

## 14. Loglama ve Denetim İzi

- Transfer başlatma correlation_id ile loglanır.

## 15. Hata Yönetimi

- Teknik hatada hata kodu + Tekrar Dene/İptal.

## 16. Bildirimler

- Para çıkışı bildirimi (Mobil Bildirim Tercihleri'ne göre).

## 17. Batch / Zamanlanmış İşler

Yoktur.

## 18. Bağımlılıklar

İşlem Onay (1.1), Kayıtlı Kişilerim (4.1), Yurt İçine (4.2), Yurt Dışına (4.3).

## 19. Açık Sorular ve Riskler

- Tüzel müşteri yetkili kişisinin işlem limiti ile müşteri limitlerinin önceliklendirmesi netleştirilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Gönderim yalnızca CustomerPersistent cüzdandan yapılabilir.
- [ ] Alt akışlar (4.1–4.3) tek transaction kaydı üzerinden yönetilir.
- [ ] İşlem İşlem Onay ekranında OTP ile onaylanır.
