# 4.3 Para Gönder — Yurt Dışına

> Ortak kurallar için bkz. `0.genel-standartlar.md`. Üst ekran: `4.para-gonder.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Para Gönder › Yurt Dışına |
| Dosya | `4.3.yurt-disina.md` |
| Ekran tipi | İşlem formu (alt akış, döviz) |
| İlgili ENUM | currency, payment_purpose, transaction_type, transaction_status |

## 2. Amaç ve Kapsam

Müşterinin yurt dışındaki bir kişiye para göndermesi alt akışıdır. Kaynak ve hedef para birimi farklı olabilir; döviz dönüşümü uygulanır.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | İşlem Başlat |
|---|---|
| Bireysel / Tüzel Müşteri | ✓ |

## 4. Menü ve Navigasyon

Para Gönder (4) menüsü altında; Kayıtlı Kişilerim (4.1) üzerinden de açılabilir. Onay için İşlem Onay (1.1).

## 5. Ekran Yapısı ve Bileşenler

**Form alanları:** Alıcı Adı ve Soyadı/Unvanı, Alıcı Ülkesi, Alıcı Telefon, Alıcı E-posta, Kaynak Para Birimi, Gönderilecek Tutar, Hedef Para Birimi, Ödeme Türü (`payment_purpose`), İşlem Açıklaması, Kişiyi Kayıtlı Kişilerime Kaydet (checkbox).

**Hesaplanan alanlar:** Dönüşüm Kuru (Kaynak + Hedef Para Birimi girilince otomatik), Alıcının Alacağı Net Tutar (Kaynak/Hedef Para Birimi + Gönderilecek Tutar girilince otomatik).

**Ücret ve Komisyonlar tablosu:** Tutar (min–max), Sabit Ücret, Oransal Ücret, Para Birimi, Kampanya Geçerli Tarih.

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| Alıcı Adı ve Soyadı / Unvanı | string | ✓ | — |
| Alıcı Ülkesi | enum | ✓ | Riskli ülke kontrolü |
| Alıcı Telefon / E-posta | string | — | — |
| Kaynak / Hedef Para Birimi | enum | ✓ | — |
| Gönderilecek Tutar | number | ✓ | Kaynak para birimi cinsinden |
| Dönüşüm Kuru / Net Tutar | number | sistem | Otomatik hesaplanır |
| Ödeme Türü | enum | ✓ | payment_purpose |
| İşlem Açıklaması | string | — | Regex pattern kontrolü |
| Kişiyi Kayıtlı Kişilerime Kaydet | bool | — | — |

## 7. Validasyon Kuralları

- Alıcı Ülkesi zorunlu; riskli ülke/ülke kısıtları uygulanır.
- Tutar limit kontrollerinden geçer.
- İşlem Açıklaması: TCKN/kart no regex engeli.

## 8. İş Kuralları ve Akışlar

- Dönüşüm Kuru ve Net Tutar otomatik hesaplanır; kur belirli süre geçerlidir, süre dolarsa uyarı + otomatik yenileme (bkz. ortak FX kuralları).
- Alıcı ülke bilgisine göre riskli ülke kısıtları uygulanır; gerekiyorsa işlem onay havuzuna düşürülür veya başlatılamaz.
- Yurt dışı statü akışı: `Pending → Sent → Completed`. `Sent` aşamasında "Partner'a iletildi, onay bekleniyor".
- "Kişiyi Kayıtlı Kişilerime Kaydet" işaretliyse alıcı 4.1'e eklenir.
- Sanction/limit/fraud kontrolleri uygulanır; İşlem özeti İşlem Onay (1.1) ekranında OTP ile onaylanır.

## 9. Statü Modeli

`transaction_status` — yurt dışı: `Pending → Sent → Completed`; hata `ErrorSend`/`ErrorReceive` (bkz. ortak statü modeli).

## 10. Veri Modeli

`transaction` (yurt dışı transfer tipi; `source_currency ≠ target_currency`, `fx_rate`, hedef ülke dolu), gerekirse `customer_saved_recipient`.

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /fx/quote` | GET | Dönüşüm kuru (süreli quote) |
| `GET /customer/fees/transfer` | GET | Ücret/komisyon tablosu |
| `POST /customer/transfers` | POST | İşlem başlat (tip: yurt dışına; idempotent) |

## 12. Entegrasyonlar

FX kuru, yurt dışı partner kuruluşu entegrasyonu, Sanction screening.

## 13. Yetkilendirme ve Güvenlik

- Yalnızca giriş yapan müşteri / işlem yetkili tüzel yetkili kişi.

## 14. Loglama ve Denetim İzi

- İşlem başlatma, kur quote, sanction sonucu correlation_id ile loglanır.

## 15. Hata Yönetimi

- Gönderim öncesi hata `ErrorSend`, sonrası `ErrorReceive`; kullanıcıya hata kodu + Tekrar Dene/İptal.

## 16. Bildirimler

- `Sent` aşamasında "Partner'a iletildi, onay bekleniyor"; para çıkışı bildirimi.

## 17. Batch / Zamanlanmış İşler

- `ErrorReceive` işlemler için retry politikası.

## 18. Bağımlılıklar

Para Gönder (4), İşlem Onay (1.1), Kayıtlı Kişilerim (4.1), Kur/FX (BackOffice 8.6).

## 19. Açık Sorular ve Riskler

- Onay havuzuna düşen yurt dışı işlemde FX quote süresi dolarsa davranış netleştirilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Dönüşüm Kuru ve Net Tutar otomatik hesaplanır.
- [ ] Riskli ülke kısıtları uygulanır.
- [ ] Yurt dışı statü akışı Pending→Sent→Completed çalışır.
- [ ] İşlem İşlem Onay ekranında OTP ile onaylanır.
