# 3. Para Yükle

> Ortak kurallar için bkz. `0.genel-standartlar.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Para Yükle |
| Dosya | `3.para-yukle.md` |
| Ekran tipi | Bilgilendirme ekranı |

## 2. Amaç ve Kapsam

Müşterinin cüzdan bakiyesini nasıl yükleyebileceğini açıklayan **bilgilendirme** ekranıdır. Müşteri uygulamasından doğrudan para yüklenemez; bakiye yükleme banka hesabından (IBAN) yapılır.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | View |
|---|---|
| Bireysel / Tüzel Müşteri | ✓ |

## 4. Menü ve Navigasyon

Sol menüde "Para Yükle". Salt bilgilendirme; işlem başlatılmaz.

## 5. Ekran Yapısı ve Bileşenler

Bilgilendirme metni: Banka hesabından (IBAN) firma hesabına havale/EFT ile para gönderilerek bakiye yükleme adımları. Açıklama alanında müşteri/cüzdan tanımlayıcısının nasıl belirtileceği anlatılır.

> İlgili müşteri IBAN'ları ve gönderim talimatları, müşteriye özel olarak gösterilebilir.

## 6. Alan Listesi

| Alan | Tip | Açıklama |
|---|---|---|
| Yükleme talimatı metni | metin | Statik/parametrik bilgilendirme |
| Hedef IBAN bilgisi | bilgi | Firma hesabı / müşteri özel referans |

## 7. Validasyon Kuralları

Bilgilendirme ekranıdır; form/validasyon yoktur.

## 8. İş Kuralları ve Akışlar

- **Müşteri uygulamasından para yüklenemez**; yükleme yalnızca banka IBAN'ı üzerinden gerçekleşir.
- Banka hesabından gelen tutar, açıklamada belirtilen müşteri/cüzdan referansına göre eşleştirilerek cüzdana yansıtılır (banka hesap hareketleri ile).

## 9. Statü Modeli

Bu ekrana özgü statü modeli yoktur.

## 10. Veri Modeli

Statik içerik; gerekirse `system_parameter` (yükleme talimat metni / IBAN).

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /customer/topup/instructions` | GET | Yükleme talimatları + IBAN bilgisi |

## 12. Entegrasyonlar

Yoktur (bakiye yansıması banka hesap hareketleri entegrasyonu ile, BackOffice tarafında).

## 13. Yetkilendirme ve Güvenlik

- Yalnızca giriş yapan müşteri.

## 14. Loglama ve Denetim İzi

- Ekran erişimi standart loglama kapsamında.

## 15. Hata Yönetimi

- Jenerik hata mesajı + kod.

## 16. Bildirimler

- Para girişi gerçekleştiğinde müşteriye bildirim (Mobil Bildirim Tercihleri'ne göre — 1.2).

## 17. Batch / Zamanlanmış İşler

Yoktur.

## 18. Bağımlılıklar

BackOffice Banka Hesap Hareketleri (6.3), Dijital Cüzdanlar (4).

## 19. Açık Sorular ve Riskler

- Müşteriye özel IBAN/referans gösteriminin (eşleştirme için) nasıl üretileceği netleştirilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Ekran yükleme talimatlarını ve IBAN bilgisini gösterir.
- [ ] Uygulamadan doğrudan para yükleme işlemi başlatılamaz.
