# 4.2 Para Gönder — Yurt İçine

> Ortak kurallar için bkz. `0.genel-standartlar.md`. Üst ekran: `4.para-gonder.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Para Gönder › Yurt İçine |
| Dosya | `4.2.yurt-icine.md` |
| Ekran tipi | İşlem formu (alt akış) |
| İlgili ENUM | currency, payment_purpose, transaction_type, transaction_status |

## 2. Amaç ve Kapsam

Müşterinin yurt içindeki bir kişiye para göndermesi alt akışıdır.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | İşlem Başlat |
|---|---|
| Bireysel / Tüzel Müşteri | ✓ |

## 4. Menü ve Navigasyon

Para Gönder (4) menüsü altında; Kayıtlı Kişilerim (4.1) üzerinden de açılabilir. Onay için İşlem Onay (1.1).

## 5. Ekran Yapısı ve Bileşenler

**Form alanları:** Alıcı Adı ve Soyadı/Unvanı, Alıcı Telefon, Alıcı E-posta, Para Birimi, Tutar, Ödeme Türü (`payment_purpose`), İşlem Açıklaması, Kişiyi Kayıtlı Kişilerime Kaydet (checkbox).

**Ücret ve Komisyonlar tablosu:** Tutar (min–max), Sabit Ücret, Oransal Ücret, Para Birimi, Kampanya Geçerli Tarih.

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| Alıcı Adı ve Soyadı / Unvanı | string | ✓ | — |
| Alıcı Telefon / E-posta | string | — | — |
| Para Birimi / Tutar | enum / number | ✓ | — |
| Ödeme Türü | enum | ✓ | payment_purpose |
| İşlem Açıklaması | string | — | Regex pattern kontrolü |
| Kişiyi Kayıtlı Kişilerime Kaydet | bool | — | İşaretliyse alıcı 4.1'e kaydedilir |

## 7. Validasyon Kuralları

- Tutar limit kontrollerinden geçer (müşteri/risk bazlı limitler; en kısıtlayıcı).
- İşlem Açıklaması: TCKN/kart no regex engeli.

## 8. İş Kuralları ve Akışlar

- Kaynak cüzdan **CustomerPersistent** olmalıdır.
- "Kişiyi Kayıtlı Kişilerime Kaydet" işaretliyse işlem sonrası alıcı `customer_saved_recipient` tablosuna eklenir.
- Sanction/limit/fraud kontrolleri uygulanır; İşlem özeti İşlem Onay (1.1) ekranında OTP ile onaylanır.

## 9. Statü Modeli

`transaction_status` — yurt içi: `Pending → Completed` (bkz. ortak statü modeli).

## 10. Veri Modeli

`transaction` (yurt içi transfer tipi), gerekirse `customer_saved_recipient`.

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /customer/fees/transfer` | GET | Ücret/komisyon tablosu |
| `POST /customer/transfers` | POST | İşlem başlat (tip: yurt içine; idempotent) |

## 12. Entegrasyonlar

Sanction screening, banka transfer (gerekirse).

## 13. Yetkilendirme ve Güvenlik

- Yalnızca giriş yapan müşteri / işlem yetkili tüzel yetkili kişi.

## 14. Loglama ve Denetim İzi

- İşlem başlatma correlation_id ile loglanır.

## 15. Hata Yönetimi

- Teknik hatada hata kodu + Tekrar Dene/İptal.

## 16. Bildirimler

- Para çıkışı bildirimi (Mobil Bildirim Tercihleri'ne göre).

## 17. Batch / Zamanlanmış İşler

Yoktur.

## 18. Bağımlılıklar

Para Gönder (4), İşlem Onay (1.1), Kayıtlı Kişilerim (4.1).

## 19. Açık Sorular ve Riskler

- Alıcının kayıtlı müşteri olup olmamasına göre fonun hangi cüzdana gideceği (Persistent/Transactional) ortak kurallara göre uygulanmalıdır.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Tutar limit kontrollerinden geçer.
- [ ] "Kayıtlı Kişilerime Kaydet" işaretliyse alıcı kaydedilir.
- [ ] İşlem İşlem Onay ekranında OTP ile onaylanır.
- [ ] Yurt içi statü akışı Pending→Completed çalışır.
