# 4.1 Para Gönder — Kayıtlı Kişilerim

> Ortak kurallar için bkz. `0.genel-standartlar.md`. Üst ekran: `4.para-gonder.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Para Gönder › Kayıtlı Kişilerim |
| Dosya | `4.1.kayitli-kisilerim.md` |
| Ekran tipi | Listeleme + form (kayıt yönetimi) |
| İlgili ENUM | payment_purpose |

## 2. Amaç ve Kapsam

Müşterinin sık para gönderdiği kişileri kaydettiği, düzenlediği ve hızlı gönderim başlattığı ekrandır.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | Gönder | Değiştir | Sil |
|---|---|---|---|---|
| Bireysel / Tüzel Müşteri | ✓ | ✓ | ✓ | ✓ |

## 4. Menü ve Navigasyon

Para Gönder (4) menüsü altında. **Gönder** butonu, yurt içi/dışı durumuna göre Yurt İçine (4.2) veya Yurt Dışına (4.3) ekranını açar.

## 5. Ekran Yapısı ve Bileşenler

**Liste kolonları:** Kayıt Adı, Adı ve Soyadı/Unvanı, Ülkesi, Telefon, E-posta.

**Butonlar:** Gönder, Değiştir, Kaydı Sil.

**Değiştir formu:** Müşteri No, Adı ve Soyadı/Unvanı, Ülkesi, Telefon, E-posta, Ödeme Türü (`payment_purpose`), İşlem Açıklaması.

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| Kayıt Adı | string | ✓ | Kullanıcı dostu etiket |
| Adı ve Soyadı / Unvanı | string | ✓ | Alıcı |
| Ülkesi | enum | ✓ | Yurt içi/dışı belirler |
| Telefon / E-posta | string | — | — |
| Müşteri No | string | — | Varsa kayıtlı müşteri |
| Ödeme Türü / İşlem Açıklaması | enum / string | — | Değiştir formunda |

## 7. Validasyon Kuralları

- Kayıt Adı ve Alıcı adı zorunlu; e-posta/telefon format kontrolü.
- İşlem Açıklaması: TCKN/kart no regex engeli.

## 8. İş Kuralları ve Akışlar

- **Gönder** → alıcı ülkesine göre Yurt İçine (4.2) veya Yurt Dışına (4.3) ekranına yönlendirir; **tutar ve para birimi dışındaki alanlar dolu**, **Alıcı bilgileri haricindeki alanlar değiştirilebilir** gelir.
- **Kaydı Sil** → onay sonrası kayıt silinir.
- **Değiştir** → form sayfası açılır; alıcı bilgileri güncellenir.

## 9. Statü Modeli

Bu ekrana özgü statü modeli yoktur (record_status ile yaşam döngüsü).

## 10. Veri Modeli

**`customer_saved_recipient`**: `id, customer_id, label, recipient_name, country, phone, email, recipient_customer_no, payment_purpose, description, created_at/by, record_status`

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /customer/saved-recipients` | GET | Liste |
| `POST /customer/saved-recipients` | POST | Yeni kayıt |
| `PUT /customer/saved-recipients/{id}` | PUT | Değiştir |
| `DELETE /customer/saved-recipients/{id}` | DELETE | Sil (soft-delete) |
