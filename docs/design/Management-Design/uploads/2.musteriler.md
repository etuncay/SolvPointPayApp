## 2. Amaç ve Kapsam

Tüm bireysel ve tüzel müşterilerin listelendiği, arandığı ve detayına gidildiği ana ekrandır. Müşteri ekleme/güncelleme alt ekranlar (2.1, 2.2) üzerinden yürür.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | View | Insert | Update | Delete | Export |
|---|---|---|---|---|---|---|
| Operasyon | ✓ | ✓ | ✓ | ✓ | — | ✓ |
| Uyum | ✓ | ✓ | — | ✓ | — | ✓ |
| Yönetim | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |

Bireysel müşteri kabulü temel olarak **temsilci** kanalından yapılır; BackOffice'te bireysel müşteri girişi/düzenlemesi yetkili personel içindir. **Tüzel müşteri kabulü yalnızca genel müdürlük** üzerinden yapılır.

## 4. Menü ve Navigasyon

- Sol menüde "Müşteriler" ana başlığı; altında 2.1–2.6.
- Satıra tıklanınca müşteri tipine göre **Yeni Bireysel Müşteri** (2.1) veya **Yeni Tüzel Müşteri** (2.2) ekranı View/Update modunda açılır.
- **Ekle** butonu müşteri tipi seçtirir ve ilgili insert ekranını açar.

## 5. Ekran Yapısı ve Bileşenler

Tek panelli grid. Varsayılan filtre: **Aktif Müşteriler**. "Tümü" seçeneği ile tüm kayıtlar listelenir.

**Kolonlar:** Müşteri No, Adı ve Soyadı / Unvanı, E-posta, Telefon, Kimlik No, Müşteri Tipi (`customer_type`), Kampanya, KYC Durumu (bireyselde `kyc_level`, tüzelde `approval_status`), Risk Skoru, Risk Segmenti, Oluşturma Tarihi, Status (`entity_status`).

Status `Blocked` veya `Closed` ise gerekçe tire ile eklenir: "Bloke – Belge Eksik", "Kapalı – Uzun Süre Kullanılmama".

## 6. Alan Listesi

| Kolon | Tip | Aranabilir | Açıklama |

| Müşteri No | string | ✓ | Tekil; 99901+ , Luhn check digit |

| Kimlik No | string | ✓ | TCKN / Pasaport / VKN |
| Müşteri Tipi | enum | ✓ | Individual / Corporate / Prospective |
| Kampanya | string | ✓ | Aktif kampanya adı |
| KYC Durumu | enum | ✓ | kyc_level / approval_status |
| Risk Skoru / Segmenti | number / enum | ✓ | Low/Medium/High/Critical |

| Status | enum | ✓ | entity_status (+gerekçe) |

## 7. Validasyon Kuralları

Listeleme ekranıdır; form validasyonu yoktur. Filtre alanlarında tip/format kontrolü uygulanır (tarih aralığı vb.).

## 8. İş Kuralları ve Akışlar

- Ekran varsayılan olarak `entity_status = Active` filtresiyle açılır.
- `record_status = 0` kayıtlar listelenmez.
- KYC Durumu kolonu müşteri tipine göre farklı ENUM gösterir (bireysel `kyc_level`, tüzel `approval_status`) — UI'da tek kolon, kaynak alan tipe bağlı.
- Excel export mevcut filtreye göre çalışır.

## 9. Statü Modeli

`entity_status`: Active / Inactive / Blocked / Closed. Detayı 2.1 ve 2.2'de.

## 10. Veri Modeli

Ana tablo `customer` (detayı 2.1/2.2). Bu ekran salt okuma yapar; kişi bilgileri `person` tablosu ile `person_id` üzerinden ilişkilidir.

**`customer`** (özet): `customer_id (PK), person_id (FK, bireysel), customer_type, campaign_id, kyc_level, kyc_status, risk_score, risk_segment, entity_status, entity_blockage_reason, entity_closure_reason, created_at/by, updated_at/by, record_status`

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /customers` | GET | Filtre + pagination ile liste |
| `GET /customers/export` | GET | Mevcut filtreye göre Excel |
