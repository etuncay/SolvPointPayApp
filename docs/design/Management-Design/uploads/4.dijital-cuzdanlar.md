# 4. Dijital Cüzdanlar

> Ortak kurallar için bkz. `0.genel-standartlar.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Dijital Cüzdanlar |
| Dosya | `4.dijital-cuzdanlar.md` |
| Ekran tipi | Listeleme (grid) |
| Detay/alt ekranlar | 4.1 Cüzdan Detay, 4.2 Cüzdan Aktiviteleri |
| İlgili ENUM | wallet_type |

## 2. Amaç ve Kapsam

Sistemdeki tüm cüzdanların (müşteri, temsilci ve sistem hesapları) listelendiği ekrandır. Detay ve hareketler 4.1 / 4.2 ekranlarında görülür.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | Görünür hesaplar |
|---|---|
| Operasyon | Müşteri / temsilci hesapları |
| Finans / Muhasebe | Reserve / Revenue / Suspense tipi sistem hesapları |
| Uyum | Gerekli görülen tüm hesaplar |

Rol bazlı görünürlük sağlanır.

## 4. Menü ve Navigasyon

Satıra tıklanınca `4.1 Cüzdan Detay` açılır.

## 5. Ekran Yapısı ve Bileşenler

Tek panelli grid. **Kolonlar:** Müşteri/Temsilci No, Hesap No, Hesap Tipi (`wallet_type`), Ad-Soyad, Telefon, Kimlik No, Kullanılabilir Bakiye, Para Birimi, Bloke Miktarı, Günlük İşlem Adedi, Günlük İşlem Tutarı, Oluşturma Tarihi.

Sistem cüzdanlarında (reserve/revenue/suspense) kişi alanları "-" gösterilir.

## 6. Alan Listesi

| Kolon | Tip | Aranabilir | Açıklama |
|---|---|---|---|
| Müşteri/Temsilci No | string | ✓ | Sistem hesabında "-" |
| Hesap No | string | ✓ | customer_id + wallet_no birlikte unique |
| Hesap Tipi | enum | ✓ | wallet_type |
| Ad-Soyad / Telefon / Kimlik No | string | ✓ | Sistem hesabında "-" |
| Kullanılabilir Bakiye | number | — | Hesaplanmış (bkz. 4.1) |
| Para Birimi | enum | ✓ | — |
| Bloke Miktarı | number | — | 0 / -1 / pozitif |
| Günlük İşlem Adedi/Tutarı | number | — | — |
| Oluşturma Tarihi | date | ✓ (aralık) | — |

## 7. Validasyon Kuralları

Listeleme ekranıdır; filtre tip/format kontrolü.

## 8. İş Kuralları ve Akışlar

- Rol bazlı satır görünürlüğü backend'de enforce edilir.
- `record_status = 0` cüzdanlar listelenmez.
- Kullanılabilir Bakiye = `Bloke Miktarı = -1` ise 0; aksi halde `max(0, Bakiye − Bloke Miktarı)`.

## 9. Statü Modeli

Cüzdana özgü ayrı statü modeli yoktur; blokaj `Bloke Miktarı` ile yönetilir (bkz. 4.1).

## 10. Veri Modeli

**`wallet`**: `id, customer_id (nullable, sistem hesabında null), agent_id (nullable), wallet_no, wallet_type, currency, balance, blocked_amount, daily_tx_count, daily_tx_amount, created_at/by, updated_at/by, record_status`

> `wallet_no` tek başına unique değildir; `customer_id + wallet_no` birlikte unique.

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /wallets` | GET | Rol bazlı filtreli liste |
| `GET /wallets/export` | GET | Excel |
| `GET /wallets/{id}` | GET | Detay (4.1) |

## 12. Entegrasyonlar

Yoktur.

## 13. Yetkilendirme ve Güvenlik

- Rol bazlı hesap tipi görünürlüğü (operasyon/finans/uyum).

## 14. Loglama ve Denetim İzi

- Cüzdan listeleme/export loglanır (finansal/PII veri erişimi).

## 15. Hata Yönetimi

- Jenerik hata mesajı + kod.

## 16. Bildirimler

Tetiklenmez.

## 17. Batch / Zamanlanmış İşler

- Günlük: Bakiye bloke bitiş tarihi gelen cüzdanların blokesinin kaldırılması (bkz. 4.1).

## 18. Bağımlılıklar

Müşteriler (2), Temsilciler (3), Para Transferleri (4.2 hareketler).

## 19. Açık Sorular ve Riskler

- Görüntülenen "Hesap No" globalde benzersiz değilse (customer_id ile birlikte unique) arama/filtreleme davranışı netleştirilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Sistem hesaplarında kişi alanları "-" gösterilir.

- [ ] Satıra tıklayınca Cüzdan Detay açılır.
