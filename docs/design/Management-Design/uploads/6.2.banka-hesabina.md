# 6.2 Para Transferi — Banka Hesabına

> Ortak kurallar için bkz. `0.genel-standartlar.md`. Üst ekran: `6.para-transferi.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Para Transferi › Banka Hesabına |
| Dosya | `6.2.banka-hesabina.md` |
| Ekran tipi | İşlem formu (alt akış) |
| İlgili ENUM | currency, payment_purpose, transaction_type, transaction_status |

## 2. Amaç ve Kapsam

Gönderen müşterinin parasını bir **banka hesabına (IBAN)** göndermesi alt akışıdır.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | İşlem Başlat |
|---|---|
| Temsilci Yetkili Kişi (işlem yetkili) | ✓ |

## 4. Menü ve Navigasyon

Para Transferi (6) menüsü altında. Onay için İşlem Onay (1.1).

## 5. Ekran Yapısı ve Bileşenler

**Form alanları:** Gönderen Müşteri No veya Kimlik No / VKN, Alıcı Adı ve Soyadı/Unvanı, Alıcı Telefon, Alıcı E-posta, IBAN, Para Birimi, Tutar, Ödeme Türü (`payment_purpose`), İşlem Açıklaması, Şüpheli İşlem Kontrolü (checkbox).

**Ücret ve Komisyonlar tablosu:** Tutar (min–max), Sabit Ücret, Oransal Ücret, Para Birimi, Kampanya Geçerli Tarih.

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| Gönderen Müşteri No / Kimlik No / VKN | string | ✓ | — |
| Alıcı Adı ve Soyadı / Unvanı | string | ✓ | — |
| Alıcı Telefon / E-posta | string | — | — |
| IBAN | string | ✓ | Max 34; TR ise 26 + checksum |
| Para Birimi / Tutar | enum / number | ✓ | — |
| Ödeme Türü | enum | ✓ | payment_purpose |
| İşlem Açıklaması | string | — | Regex pattern kontrolü |
| Şüpheli İşlem Kontrolü | bool | — | — |

## 7. Validasyon Kuralları

- IBAN format + checksum; **IBAN "TR" ile başlamıyorsa yurt dışı akışına (6.4) yönlendirilir.**
- İşlem Açıklaması: TCKN/kart no regex engeli.

## 8. İş Kuralları ve Akışlar

- IBAN TR değilse otomatik yurt dışı akışına yönlendirme.
- Banka hesabına giden işlemlerde **Alıcı Cüzdan boş**, IBAN dolu olur.
- Üst ekran (6) sorgu, sanction ve idempotency kuralları geçerlidir.
- İşlem özeti İşlem Onay (1.1) ekranında onaylanır.

## 9. Statü Modeli

`transaction_status` — bkz. ortak statü modeli.

## 10. Veri Modeli

`transaction` (banka transferi tipi; `iban` dolu, `receiver_wallet_id` boş).

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /agent/fees/transfer` | GET | Ücret/komisyon tablosu |
| `POST /agent/transfers` | POST | İşlem başlat (tip: banka hesabına) |
| `GET /iban/validate` | GET | IBAN doğrulama |

## 12. Entegrasyonlar

IBAN doğrulama, banka transfer (EFT/FAST), Sanction screening.

## 13. Yetkilendirme ve Güvenlik

- İşlem yetkili temsilci yetkili kişisi.

## 14. Loglama ve Denetim İzi

- İşlem başlatma correlation_id ile loglanır.

## 15. Hata Yönetimi

- Teknik hatada hata kodu + Tekrar Dene/İptal.

## 16. Bildirimler

- İşlem sonucu ilgili taraflara bildirim (parametrik).

## 17. Batch / Zamanlanmış İşler

Yoktur.

## 18. Bağımlılıklar

Para Transferi (6), İşlem Onay (1.1), Yurt Dışına (6.4 — yurt dışı IBAN), Bankalar (BackOffice 6).

## 19. Açık Sorular ve Riskler

- Yurt dışı IBAN tespitinde yalnızca "TR" ön eki yeterli mi, ülke kodu doğrulaması gerekir mi netleştirilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] IBAN checksum kontrolü çalışır.
- [ ] TR olmayan IBAN yurt dışı akışına yönlendirir.
- [ ] Banka hesabına işlemde Alıcı Cüzdan boş, IBAN dolu kaydedilir.
- [ ] İşlem İşlem Onay ekranında onaylanır.
