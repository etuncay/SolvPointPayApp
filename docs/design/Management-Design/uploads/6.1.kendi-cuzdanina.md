# 6.1 Para Transferi — Kendi Cüzdanına

> Ortak kurallar için bkz. `0.genel-standartlar.md`. Üst ekran: `6.para-transferi.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Para Transferi › Kendi Cüzdanına |
| Dosya | `6.1.kendi-cuzdanina.md` |
| Ekran tipi | İşlem formu (alt akış) |
| İlgili ENUM | currency, transaction_type, transaction_status |

## 2. Amaç ve Kapsam

Müşterinin temsilci lokasyonunda nakit getirerek **kendi cüzdanına** bakiye yüklemesi (para yatırma) alt akışıdır. Hedef cüzdan gönderen müşteriye aittir.

> **Not:** Bu akış, müşterinin getirdiği nakdin kendi cüzdanına aktarılmasıdır; "cüzdandan transfer" yasağı (temsilci kanalı) ile çelişmez — fonun kaynağı temsilcideki nakittir.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | İşlem Başlat |
|---|---|
| Temsilci Yetkili Kişi (işlem yetkili) | ✓ |

## 4. Menü ve Navigasyon

Para Transferi (6) menüsü altında. Onay için İşlem Onay (1.1).

## 5. Ekran Yapısı ve Bileşenler

**Form alanları:** Gönderen Müşteri No veya Kimlik No / VKN, Para Birimi, Tutar, Cüzdan Numarası (combobox — Müşteri No ve para birimi dolunca otomatik dolar, değiştirilemez), Şüpheli İşlem Kontrolü (checkbox).

**Ücret ve Komisyonlar tablosu:** Tutar (min–max), Sabit Ücret, Oransal Ücret, Para Birimi, Kampanya Geçerli Tarih.

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| Gönderen Müşteri No / Kimlik No / VKN | string | ✓ | — |
| Para Birimi | enum | ✓ | — |
| Tutar | number | ✓ | Yatırılacak nakit tutarı |
| Cüzdan Numarası | enum | sistem | Otomatik dolar, değiştirilemez |
| Şüpheli İşlem Kontrolü | bool | — | — |

## 7. Validasyon Kuralları

- Hedef cüzdan **aynı müşteriye ait** olmalıdır.
- Para yatırma için müşteri KYC seviyesi şartını sağlamalıdır (asgari Level 2 — temsilciden kendi cüzdanına yatırma; bkz. ortak KYC kuralları).

## 8. İş Kuralları ve Akışlar

- Müşteri No + Para Birimi dolduğunda Cüzdan Numarası otomatik gelir.
- Üst ekran (6) sorgu, sanction ve idempotency kuralları geçerlidir.
- İşlem özeti İşlem Onay (1.1) ekranında onaylanır.

## 9. Statü Modeli

`transaction_status` — bkz. ortak statü modeli.

## 10. Veri Modeli

`transaction` (kendi cüzdanına yatırma tipi), `wallet`.

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /agent/customers/{id}/wallets` | GET | Müşteri cüzdanları |
| `GET /agent/fees/transfer` | GET | Ücret/komisyon tablosu |
| `POST /agent/transfers` | POST | İşlem başlat (tip: kendi cüzdanına) |

## 12. Entegrasyonlar

Sanction screening, Doküman Yönetim Sistemi.

## 13. Yetkilendirme ve Güvenlik

- İşlem yetkili temsilci yetkili kişisi.

## 14. Loglama ve Denetim İzi

- İşlem başlatma correlation_id ile loglanır.

## 15. Hata Yönetimi

- Teknik hatada hata kodu + Tekrar Dene/İptal.

## 16. Bildirimler

- İşlem sonucu müşteriye bildirim (parametrik).

## 17. Batch / Zamanlanmış İşler

Yoktur.

## 18. Bağımlılıklar

Para Transferi (6), İşlem Onay (1.1).

## 19. Açık Sorular ve Riskler

- Para yatırma için minimum KYC seviyesi (Level 2) UI'da net kontrol edilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Cüzdan Numarası otomatik dolar ve değiştirilemez.
- [ ] Hedef cüzdan aynı müşteriye ait olmalıdır.
- [ ] Ücret/komisyon tablosu doğru gösterilir.
- [ ] İşlem İşlem Onay ekranında onaylanır.
