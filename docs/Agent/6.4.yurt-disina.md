# 6.4 Para Transferi — Yurt Dışına

> Ortak kurallar için bkz. `0.genel-standartlar.md`. Üst ekran: `6.para-transferi.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Para Transferi › Yurt Dışına |
| Dosya | `6.4.yurt-disina.md` |
| Ekran tipi | İşlem formu (alt akış, döviz) |
| İlgili ENUM | currency, payment_purpose, transaction_type, transaction_status |

## 2. Amaç ve Kapsam

Gönderen müşterinin parasını **yurt dışına** göndermesi alt akışıdır. Kaynak ve hedef para birimi farklı olabilir; döviz dönüşümü uygulanır.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | İşlem Başlat |
|---|---|
| Temsilci Yetkili Kişi (işlem yetkili) | ✓ |

## 4. Menü ve Navigasyon

Para Transferi (6) menüsü altında. Onay için İşlem Onay (1.1).

## 5. Ekran Yapısı ve Bileşenler

**Form alanları:** Gönderen Müşteri No veya Kimlik No / VKN, Alıcı Adı ve Soyadı/Unvanı, Alıcı Ülkesi, Alıcı Telefon, Alıcı E-posta, Kaynak Para Birimi, Gönderilecek Tutar, Hedef Para Birimi, Ödeme Türü (`payment_purpose`), İşlem Açıklaması, Şüpheli İşlem Kontrolü (checkbox).

**Hesaplanan alanlar:** Dönüşüm Kuru (Kaynak + Hedef Para Birimi girilince otomatik), Alıcının Alacağı Net Tutar (Kaynak/Hedef Para Birimi + Gönderilecek Tutar girilince otomatik). "Bu kur X saniye geçerlidir." (X parametreden).

**Ücret ve Komisyonlar tablosu:** Tutar (min–max), Sabit Ücret, Oransal Ücret, Para Birimi, Kampanya Geçerli Tarih.

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| Gönderen Müşteri No / Kimlik No / VKN | string | ✓ | — |
| Alıcı Adı ve Soyadı / Unvanı | string | ✓ | — |
| Alıcı Ülkesi | enum | ✓ | Riskli ülke kontrolü |
| Alıcı Telefon / E-posta | string | — | — |
| Kaynak / Hedef Para Birimi | enum | ✓ | — |
| Gönderilecek Tutar | number | ✓ | Kaynak para birimi cinsinden |
| Dönüşüm Kuru / Net Tutar | number | sistem | Otomatik hesaplanır |
| Ödeme Türü | enum | ✓ | payment_purpose |
| İşlem Açıklaması | string | — | Regex pattern kontrolü |

## 7. Validasyon Kuralları

- Alıcı Ülkesi seçimi zorunlu; riskli ülke/ülke kısıtları uygulanır.
- İşlem Açıklaması: TCKN/kart no regex engeli.

## 8. İş Kuralları ve Akışlar

- Dönüşüm Kuru ve Net Tutar otomatik hesaplanır. **Kur X saniye geçerlidir**; süre dolarsa kullanıcıya uyarı verilir, kur otomatik yenilenir ve net tutar yeniden hesaplanır.
- Alıcı ülke bilgisine göre **riskli ülke/ülke kısıtları** uygulanır; gerekiyorsa işlem onay havuzuna düşürülür veya işlem başlatılamaz (kural/limit parametrelerine göre).
- Yurt dışı işlem statü akışı: `Pending → Sent → Completed`. `Sent` aşamasında ekranda "Partner'a iletildi, onay bekleniyor" mesajı.
- Üst ekran (6) sorgu, sanction ve idempotency kuralları geçerlidir.
- İşlem özeti İşlem Onay (1.1) ekranında onaylanır.

## 9. Statü Modeli

`transaction_status` — yurt dışı: `Pending → Sent → Completed`; hata `ErrorSend`/`ErrorReceive` (bkz. ortak statü modeli).

## 10. Veri Modeli

`transaction` (yurt dışı transfer tipi; `source_currency ≠ target_currency`, `fx_rate`, hedef ülke dolu).

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /fx/quote` | GET | Dönüşüm kuru (süreli quote) |
| `GET /agent/fees/transfer` | GET | Ücret/komisyon tablosu |
| `POST /agent/transfers` | POST | İşlem başlat (tip: yurt dışına) |
| `POST /sanction/screen` | POST | Sanction taraması |

## 12. Entegrasyonlar

FX kuru, yurt dışı partner kuruluşu entegrasyonu, Sanction screening.

## 13. Yetkilendirme ve Güvenlik

- İşlem yetkili temsilci yetkili kişisi.

## 14. Loglama ve Denetim İzi

- İşlem başlatma, kur quote, sanction sonucu correlation_id ile loglanır.

## 15. Hata Yönetimi

- Gönderim öncesi hata `ErrorSend`, sonrası `ErrorReceive`; kullanıcıya hata kodu + Tekrar Dene/İptal.
- `ErrorReceive → Retrying` sürecinde dış sistemde kaydın varlığı kontrol edilir (mükerrer engelleme).

## 16. Bildirimler

- `Sent` aşamasında "Partner'a iletildi, onay bekleniyor"; sonuç bildirimi (parametrik).

## 17. Batch / Zamanlanmış İşler

- `ErrorReceive` işlemler için retry politikası (parametrik deneme sayısı/süre).

## 18. Bağımlılıklar

Para Transferi (6), İşlem Onay (1.1), Kur/FX Yönetimi (BackOffice 8.6), Risk referans verileri (BackOffice 7.6).

## 19. Açık Sorular ve Riskler

- İşlem onay havuzuna düşen yurt dışı işlemde FX quote süresi dolarsa davranış netleştirilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Dönüşüm Kuru ve Net Tutar otomatik hesaplanır.
- [ ] Kur süresi dolunca uyarı + otomatik yenileme çalışır.
- [ ] Riskli ülke kısıtları uygulanır.
- [ ] Yurt dışı statü akışı Pending→Sent→Completed çalışır.
- [ ] ErrorReceive retry'da mükerrer gönderim engellenir.
