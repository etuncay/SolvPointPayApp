# 5. Para Çekme

> Ortak kurallar için bkz. `0.genel-standartlar.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Para Çekme |
| Dosya | `5.para-cekme.md` |
| Ekran tipi | İşlem formu (sorgu + form + onay) |
| İlgili ENUM | identity_document, currency, transaction_type, transaction_status |

## 2. Amaç ve Kapsam

Alıcı kişinin temsilci lokasyonundan **nakit para çekmesi** (cash payout) işlemini yürüten ekrandır. Ekrandaki müşteri bilgileri **alıcıya** aittir.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | İşlem Başlat | Onayla |
|---|---|---|
| Temsilci Yetkili Kişi (işlem yetkili) | ✓ | ✓ (İşlem Onay — 1.1) |

## 4. Menü ve Navigasyon

Sol menüde "Para Çekme". İşlem özeti için İşlem Onay (1.1) ekranına; müşteri eksikse Yeni Bireysel Müşteri Kaydı (3) ekranına yönlendirir.

## 5. Ekran Yapısı ve Bileşenler

**Form alanları:** Müşteri No veya Kimlik No / VKN, İşlem Referans No, Yurt Dışı Referans No, Adı ve Soyadı/Unvanı, Para Birimi (combobox), Tutar, Şüpheli İşlem Kontrolü (checkbox). İşlem Referans No veya Yurt Dışı Referans No'dan biri yeterlidir.

**Ücret ve Komisyonlar tablosu:** Tutar (min–max), Sabit Ücret, Oransal Ücret, Para Birimi, Kampanya Geçerli Tarih.

**Kimlik tarama alanları** (sorgu sonrası): Kimlik Ülkesi/Uyruğu, Kimlik Tipi, Doğum Tarihi, Kimlik Belgesi Ön/Arka + Belge Yükle butonu.

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| Müşteri No / Kimlik No / VKN | string | ✓ | Alıcı sorgu anahtarı |
| İşlem / Yurt Dışı Referans No | string | koşullu | Biri yeterli |
| Para Birimi | enum | ✓ | Cüzdan tek ise otomatik |
| Tutar | number | koşullu | Cüzdan türüne göre belirlenir |
| Şüpheli İşlem Kontrolü | bool | — | Bilgi kutusu: "Sadece şüpheli işlemler için işaretleyiniz." |

## 7. Validasyon Kuralları

- Kimlik Ülkesi = TUR ise Kimlik Tipi = `IdentityCard` zorunlu, değiştirilemez.
- Tutar/Para Birimi cüzdan kurallarına uygun olmalı (aşağıda).

## 8. İş Kuralları ve Akışlar

- **Bireysel müşteri** aktif ve `kyc_level ≥ 1` ise: kimlik belgesi ön/arka taranıp eklenir. Taranmış kimlik OCR ile sistemdeki bilgilerle karşılaştırılır (**KPS kontrolü yapılmaz**).
- **Tüzel müşteri** aktif ve `kyc_status = Approved` ise: Yetkili Kişi Kimlik No alanı görünür; kişinin firma adına para çekme yetkisi kontrol edilir. Tüzel kişinin eksik/süresi geçmiş belgesi varsa "İşleminizin devam etmesi için müşteri belgelerindeki eksiklikler tamamlanmalıdır..." uyarısı + Belge Ekle.
- **Müşteri tanımı yoksa** veya eksik bilgi/belge varsa (Incomplete or Incorrect Information / Missing Documents) → "Yeni Müşteri Kaydı ekranına yönlendiriliyorsunuz." Kayıt sonrası aynı Para Çekme ekranına dönülür, sorgu yenilenir.
- **Cüzdan/Tutar:** Tek cüzdan varsa Para Birimi otomatik. Tutar: `CustomerTransactional` ise kullanılabilir bakiye otomatik + değiştirilemez (**tüm bakiye çekilir**); `CustomerPersistent` ise varsayılan gelir ama değiştirilebilir. Çoklu cüzdanda Para Birimi seçimi sonrası önce o para birimindeki Transactional, yoksa Persistent cüzdan kullanılır.
- **Sanction:** Para çekme öncesi `kyc_level ≤ 1` gerçek kişiler için kara liste/sanction kontrolü; hit halinde işlem `OnHold`, kişi `Blocked`, KYC Yönetimi'nde inceleme kaydı açılır.
- **Idempotency:** Aynı İşlem Referans No ile tekrar denemede "İşlem daha önce başlatılmış/tamamlanmış" uyarısı; ikinci ödeme yaratılmaz.
- İşlem özeti İşlem Onay (1.1) ekranında onaylanır.

## 9. Statü Modeli

`transaction_status` — bkz. ortak para transferi statü modeli.

## 10. Veri Modeli

`transaction` (para çekme tipi), `wallet`, `customer`, `dms.document` (Identity).

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /agent/customers/search` | GET | Alıcı sorgu |
| `GET /agent/fees/withdrawal` | GET | Ücret/komisyon tablosu |
| `POST /agent/withdrawals` | POST | Para çekme başlat (idempotent) |
| `POST /sanction/screen` | POST | Sanction taraması |

## 12. Entegrasyonlar

OCR, Sanction screening, Doküman Yönetim Sistemi.

## 13. Yetkilendirme ve Güvenlik

- İşlem yetkili temsilci yetkili kişisi; OTP onayı İşlem Onay'da.

## 14. Loglama ve Denetim İzi

- Para çekme başlatma, sanction sonucu, idempotency olayları correlation_id ile loglanır.

## 15. Hata Yönetimi

- Teknik hatada hata kodu + Tekrar Dene/İptal (bkz. `0.6`).

## 16. Bildirimler

- İşlem sonucu ilgili taraflara bildirim (parametrik).

## 17. Batch / Zamanlanmış İşler

Yoktur.

## 18. Bağımlılıklar

İşlem Onay (1.1), Yeni Bireysel Müşteri Kaydı (3), Müşteri Ücret/Kampanya (BackOffice 2.5/2.6).

## 19. Açık Sorular ve Riskler

- CustomerTransactional cüzdandan tüm bakiyenin çekilmesi zorunluluğu UI'da net gösterilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] CustomerTransactional cüzdanda tutar tüm bakiye + değiştirilemez.
- [ ] Müşteri eksikse Yeni Müşteri Kaydı'na yönlendirir, dönüşte sorgu yenilenir.
- [ ] Level ≤1 kişide sanction kontrolü çalışır; hit'te işlem OnHold olur.
- [ ] Aynı referans ile mükerrer ödeme engellenir.
- [ ] Tüzel müşteride yetkili kişi yetki kontrolü yapılır.
