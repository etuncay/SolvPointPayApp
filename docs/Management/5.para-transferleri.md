# 5. Para Transferleri

> Ortak kurallar için bkz. `0.genel-standartlar.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Para Transferleri |
| Dosya | `5.para-transferleri.md` |
| Ekran tipi | Listeleme (grid) |
| Detay/alt ekranlar | 5.1 İşlem Detay, 5.2 İade / İptal / Düzeltme |
| İlgili ENUM | transaction_type, transaction_status |

## 2. Amaç ve Kapsam

Tüm para hareketlerinin (yurt içi/yurt dışı transfer, para yatırma, para çekme) listelendiği ve izlendiği ekrandır. İşlem girişi temsilci/müşteri uygulamalarında yapılır; BackOffice bu işlemleri izler ve müdahale eder (bloke/iptal).

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | View | Export | Bloke/İptal |
|---|---|---|---|---|
| Operasyon | ✓ | ✓ | ✓ | yetkiye bağlı |
| Uyum | ✓ | ✓ | ✓ | ✓ |
| Finans | ✓ | ✓ | ✓ | — |

## 4. Menü ve Navigasyon

Satıra tıklanınca `5.1 İşlem Detay` açılır. `5.2 İade/İptal/Düzeltme` manuel transfer formudur.

## 5. Ekran Yapısı ve Bileşenler

Tek panelli grid. **Kolonlar:** İşlem Numarası, İşlem Tarihi, Gönderen Müşteri No, Gönderen Cüzdan, Gönderen Adı ve Soyadı / Unvanı, Alıcı Müşteri No, Alıcı Cüzdan, Alıcı Adı ve Soyadı / Unvanı, Gönderen Temsilci No, Alıcı Temsilci No, IBAN, İşlem Türü, Para Birimi, Tutar, Status (`transaction_status`).

## 6. Alan Listesi

| Kolon | Tip | Aranabilir | Açıklama |
|---|---|---|---|
| İşlem Numarası | string | ✓ | — |
| İşlem Tarihi | datetime | ✓ (aralık) | — |
| Gönderen/Alıcı Müşteri No, Cüzdan, Ad | string | ✓ | Kaynak/hedef taraf |
| Gönderen/Alıcı Temsilci No | string | ✓ | — |
| IBAN | string | ✓ | Yalnızca bankaya giden/IBAN içeren akışlarda dolu |
| İşlem Türü / Para Birimi | enum | ✓ | — |
| Tutar | number | — | **Kaynak tutar** esas alınır; hedef tutar detayda |
| Status | enum | ✓ | transaction_status |

## 7. Validasyon Kuralları

Listeleme ekranıdır; filtre tip/format kontrolü.

## 8. İş Kuralları ve Akışlar

- Tutar/Para Birimi listede **Kaynak tutar** ile gösterilir; hedef tutar/para birimi İşlem Detay'da.
- `record_status = 0` işlemler listelenmez (soft-update yeni satır mantığı).
- Excel export filtreye göre çalışır.

## 9. Statü Modeli

`transaction_status` — bkz. `0.9` (Pending, Sent, Completed, OnHold, Unblocked, Canceled, ErrorComplete, ErrorSend, ErrorReceive, Retrying).

## 10. Veri Modeli

**`transaction`** (özet): `transaction_id (PK), correlation_id, reference_no, foreign_reference_no, sender_customer_id, sender_wallet_id, receiver_customer_id, receiver_wallet_id, sender_agent_id, receiver_agent_id, iban, transaction_type, payment_purpose, source_currency, target_currency, principal_amount, fee_fixed, fee_variable, fx_rate, status, channel, created_at/by, ..., record_status`

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /transactions` | GET | Filtre + pagination |
| `GET /transactions/export` | GET | Excel |
| `GET /transactions/{id}` | GET | İşlem detay (5.1) |

## 12. Entegrasyonlar

Dolaylı: muhasebe, banka, partner entegrasyonları (işlem yaşam döngüsünde).

## 13. Yetkilendirme ve Güvenlik

- Bloke/iptal yetki bazlı (5.1 üzerinden).

## 14. Loglama ve Denetim İzi

- Listeleme/export finansal veri erişimi olarak loglanır.

## 15. Hata Yönetimi

- Jenerik hata mesajı + kod.

## 16. Bildirimler

Tetiklenmez (bildirimler işlem akışında ve 5.1'de).

## 17. Batch / Zamanlanmış İşler

- `ErrorReceive` işlemlerde retry politikası (parametrik deneme sayısı/süre).

## 18. Bağımlılıklar

İşlem Detay (5.1), İade/İptal/Düzeltme (5.2), Dijital Cüzdanlar (4), Bankalar (6), Muhasebe Entegrasyon (8.3).

## 19. Açık Sorular ve Riskler

- Statü modeli `Pending/Error* → OnHold` geçişine izin verirken İşlem Detay ekranı "Bloke Et" butonunu yalnızca Pending'de gösteriyor; Error* → OnHold yetkisinin UI karşılığı netleştirilmeli (bkz. `0.19`).
