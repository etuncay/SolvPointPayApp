# 3.1 Yeni Temsilci

> Ortak kurallar için bkz. `0.genel-standartlar.md`. Insert/Update/View ekranı aynıdır.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Temsilciler › Yeni Temsilci |
| Dosya | `3.1.yeni-temsilci.md` |
| Ekran tipi | Tek sayfa form — Insert/Update/View |
| İlgili ENUM | organization_type_commercial, ownership_type, settlement_frequency, entity_status, level, address_type, status |

## 2. Amaç ve Kapsam

Temsilci (acente) kaydının oluşturulduğu, güncellendiği ve görüntülendiği ekrandır. Temsilciler **yalnızca tüzel kişilik** olabilir; KYC süreçleri tüzel kişilerle aynı yürür. Temsilci kabulü yalnızca Genel Müdürlük tarafından yapılır.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | Insert | Update | View | Bloke Et/Kaldır |
|---|---|---|---|---|
| Genel Müdürlük Operasyon | ✓ | ✓ | ✓ | yetkiye bağlı |
| Uyum | — | ✓ | ✓ | ✓ |
| Yönetim | ✓ | ✓ | ✓ | ✓ |

## 4. Menü ve Navigasyon

Temsilciler (3) listesinden Ekle veya satır seçimi ile açılır. Belge Görme/İndirme → DMS (9).

## 5. Ekran Yapısı ve Bileşenler

**Butonlar:** Temsilci Aktiviteleri, Bloke Et, Bloke Kaldır, Belge Yükleme, Belge Görme/İndirme, Taslak Kaydet, Kaydet, Vazgeç.

**Paneller:**

1. **Temsilci Bilgileri Paneli** — Temsilci No (değiştirilemez), Ticari Unvan, Vergi Kimlik No, Temsilci Grubu (combobox), Komisyon Mahsuplaşma (`settlement_frequency`), Oluşturma Tarihi, Risk Segmenti, Status (+gerekçe).
2. **Banka Hesapları Paneli** — Banka, IBAN, Para Birimi, Şube Kodu, Hesap No, Ek No, Varsayılan Hesap Mı, Status. Yabancı banka hesabı girilemez.
3. **Finansal Bilgiler Paneli** — No, Tür (Avans / Komisyon), Bakiye, Para Birimi, Bloke Miktarı, Günlük İşlem Adedi/Tutarı (salt-okunur).
4. **Tüzel Kişi Paneli** — Hukuki Statü (`organization_type_commercial`), Sicil Numarası, Mersis Numarası, Vergi Dairesi, Faaliyet Konusu, Kuruluş Tarihi, Kurulduğu Ülke, Sermaye Yapısı, Personel Sayısı, Dil Tercihi, Notlar.
5. **Ortaklar Paneli** — 2.2 ile aynı kurallar (TCKN/VKN, Tür, Pay, UBO, tarih).
6. **Yetkililer Paneli** — 2.2 ile aynı kurallar (İşlem/İmza Yetkisi, limitler).
7. **Adres Paneli** — çoklu adres; Türkiye için Enlem/Boylam alanları da alınır (fraud mesafe hesabı için); en az bir `Registered` adres.
8. **İletişim Bilgileri Paneli** — E-posta/Telefon; doğrulama; asıl kanal.
9. **İşlem Limitleri Paneli** — İşlem Limiti, Günlük İşlem Limiti, Aylık İşlem Limiti.

## 6. Alan Listesi

Alanlar panellerde listelenmiştir. VKN girilince mevcut temsilci varsa ekran o bilgilerle dolar. Adres panelinde **Enlem/Boylam** temsilciye özgüdür (impossible-travel/mesafe kuralları için).

## 7. Validasyon Kuralları

| Alan | Kontrol |
|---|---|
| Adı ve Soyadı / Unvanı | Boş olamaz; 2–255 karakter |
| Vergi Kimlik No | 10 hane rakam; ilk hane ≠ 0 |
| IBAN | Max 34; TR ise 26 + checksum; mükerrer yok |
| Hukuki Statü | Seçim zorunlu (organization_type_commercial) |
| Sicil Numarası | OrdinaryPartnership hariç zorunlu; 5–30 karakter |
| MERSİS Numarası | SoleProprietorship, OrdinaryPartnership hariç zorunlu; 16 hane |
| Vergi Dairesi / Kurulduğu Ülke / Sermaye Yapısı / Dil | Seçim zorunlu |
| Faaliyet Konusu | 3–255 karakter |
| Kuruluş Tarihi | OrdinaryPartnership hariç zorunlu; gelecek tarih olamaz |
| Personel Sayısı | Pozitif tam sayı |
| Adres / İletişim | bkz. 2.1 kuralları; ayrıca en az 1 irtibat adresi |

## 8. İş Kuralları ve Akışlar

- **Kaydet akışı:** zorunlu alan → **yetkili KYC seviyesi** kontrolü → belge kontrolü (Hukuki Statü zorunlu belge seti eksiksiz, tümü `approval_required=false` veya `Approved` — zorunlu belge eşlemesi için bkz. `0.genel-standartlar.md` §0.19 *Tüzel Kişiler – Zorunlu Belgeler*) → arka planda sanction/kara liste taraması. Asgari belge seti eksikse Kaydet bloklanır ve eksik belge(ler) kullanıcıya açıkça bildirilir.
- Temsilci adına işlem için `kyc_status=Approved` ve `entity_status=Active`; işlemi yapan yetkili Yetkililer panelinde tanımlı ve `status=Active` olmalı; aksi halde **backend seviyesinde reddedilir**.
- **KYC seviyesi şartı (temsilci özel):** Temsilcide müşteri verilerine erişim olduğundan, **işlem yapan VE sadece görüntüleme yetkili tüm yetkililer için `kyc_level ≥ 2`** aranır. UBO'lar için `kyc_level ≥ 1`. UBO kaydı/seviyesi eksikse temsilci işlem yapamaz ve temsilci süreçleri tamamen durdurulur.
- **%25 / UBO zinciri** kuralları 2.2 ile aynıdır.
- Mevcut temsilci güncellemede Tüzel Kişi / Ortaklar / Yetkililer / Adres değişikliğinde tüm kontroller yeniden işletilir.
- Bloke Et/Kaldır 2.1 ile aynı (tam bloke; bakiye blokesi 4.1'den).
- Taslak kayıt → Inactive.
- Sisteme yeni eklenen her temsilci otomatik **varsayılan temsilci grubuna** atanır (bkz. 3.3).

## 9. Statü Modeli

`kyc_status`: NULL → Pending → Approved/Rejected. `entity_status`: Inactive → Active → Blocked → Active/Closed. KYB akışı `2.2` tablosuyla aynıdır.

## 10. Veri Modeli

- **`agent`**: bkz. `3.temsilciler.md` + `agent_group_id, settlement_frequency`.
- **`agent_shareholder`**, **`agent_authorized_person`** (yapı `corporate_shareholder` / `corporate_authorized_person` ile aynı).
- **`agent_bank_account`**, **`agent_address`** (+ latitude, longitude), **`agent_contact`**, **`agent_transaction_limit`**.
- Avans/Komisyon hesapları `wallet` tablosunda `AgentAdvance` / `AgentCommission` tipiyle.

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /agents/{id}` | GET | Detay |
| `POST /agents` | POST | Yeni kayıt |
| `PUT /agents/{id}` | PUT | Güncelle |
| `POST /agents/draft` | POST | Taslak |
| `POST /agents/{id}/block` / `unblock` | POST | Bloke |
| `POST /sanction/screen` | POST | Sanction (temsilci + UBO + yetkili) |

## 12. Entegrasyonlar

Ticaret Sicil / MERSİS / VKN doğrulama, Sanction screening, Doküman Yönetim Sistemi.

## 13. Yetkilendirme ve Güvenlik

- Temsilci kabulü yalnızca Genel Müdürlük rolleri.
- Bloke Et/Kaldır yetki bazlı, loglanır.

## 14. Loglama ve Denetim İzi

- Insert/Update/Bloke eski-yeni değer ile loglanır; ortaklık/yetkili değişiklikleri ayrı loglanır.

## 15. Hata Yönetimi

- MERSİS/sanction erişilemezse arka plan kontrolü beklemede; belge eksikse Kaydet bloklanır.

## 16. Bildirimler

- İletişim bilgisi değişiminde eski adrese bilgilendirme; telefon OTP / e-posta doğrulama.

## 17. Batch / Zamanlanmış İşler

- Aylık: Periyodik sanction taraması (temsilci + UBO + yetkili).
- Günlük: Bloke bitiş tarihi gelen temsilcilerin Active yapılması.

## 18. Bağımlılıklar

`person`, Temsilci Grup Yönetimi (3.3), DMS (9), Doküman Tipleri (9.3), Tüzel Kişiler – Zorunlu Belgeler tablosu (`0.genel-standartlar.md` §0.19), KYC Yönetimi (8.2), Dijital Cüzdanlar (4).

## 19. Açık Sorular ve Riskler

- Yetkili kişi limitlerinin Temsilci uygulamasındaki "Kullanıcı Yönetimi" ekranıyla çakışma riski; tek kaynak/öncelik kuralı tanımlanmalı.
- Read-only yetkili için bile `kyc_level ≥ 2` şartı — Temsilci başlatma maliyetini artırır; iş tarafıyla teyit edilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Temsilci yalnızca tüzel kişilik olarak kaydedilebilir.
- [ ] Tüm yetkililer için `kyc_level ≥ 2` şartı uygulanır.
- [ ] UBO eksikse temsilci süreçleri durdurulur.
- [ ] Yeni temsilci varsayılan gruba otomatik atanır.
- [ ] Adres panelinde Enlem/Boylam alınır.
- [ ] Kaydet'te belge ve sanction kontrolleri çalışır.
