# 2.6 Müşteri Kampanya Yönetimi

> Ortak kurallar için bkz. `0.genel-standartlar.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Müşteriler › Müşteri Kampanya Yönetimi |
| Dosya | `2.6.musteri-kampanya-yonetimi.md` |
| Ekran tipi | Listeleme + satır içi ekleme/güncelleme (detay sayfası yok) |
| İlgili ENUM | transaction_type, currency, status |

## 2. Amaç ve Kapsam

Müşterilere uygulanacak kampanyaların yönetildiği ekrandır. Kampanyalar, standart müşteri ücret tarifesi (2.5) üzerine **ayarlama (indirim/ücretsiz/avantaj)** uygular.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | Insert | Update | Export |
|---|---|---|---|---|
| Pazarlama / Fiyatlandırma | ✓ | ✓ | ✓ | ✓ |
| Operasyon | ✓ | — | — | ✓ |

## 4. Menü ve Navigasyon

Müşteriler menüsü altında; satır içi ekleme/güncelleme.

## 5. Ekran Yapısı ve Bileşenler

Tek tablo. **Kolonlar:** Kampanya Kodu, Kampanya Adı, Açıklama, Sabit Ücret Kazanç Oranı, Komisyon Kazanç Oranı, İşlem (`transaction_type`), Para Birimi (`currency`), Başlangıç Tarihi, Bitiş Tarihi, Minimum İşlem Tutarı, Maksimum Kazanç – İşlem Başı, Maksimum Kazanç – Toplam, Maksimum Kullanım Adedi, Status (`status`).

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| Kampanya Kodu | string | ✓ | Sistemsel anahtar; update'te değiştirilemez |
| Kampanya Adı / Açıklama | string | ✓ / — | — |
| Sabit Ücret / Komisyon Kazanç Oranı | number | ✓ | Standart tarifeye uygulanacak ayarlama |
| İşlem / Para Birimi | enum | ✓ | — |
| Başlangıç / Bitiş Tarihi | date | — | — |
| Minimum İşlem Tutarı | number | — | Doluysa kampanya bu tutar ve üzeri işlemlerde |
| Maks. Kazanç – İşlem Başı | number | — | Tek işlemde sağlanabilecek en yüksek avantaj |
| Maks. Kazanç – Toplam | number | — | Kampanya süresince müşteri başına toplam avantaj |
| Maks. Kullanım Adedi | int | — | Müşteri başına azami işlem adedi |
| Status | enum | sistem | Active/Passive |

## 7. Validasyon Kuralları

- Kampanya Kodu tekildir; update ekranında değiştirilemez (değişiklik için yeni kampanya).
- Oran/tutar alanları negatif olamaz; tarih formatı geçerli olmalı.

## 8. İş Kuralları ve Akışlar

- **Bir müşteriye aynı anda birden fazla kampanya tanımlanamaz.** Yeni kampanya tanımlanırsa kullanıcıdan validasyon alınır, mevcut aktif kayıt pasife alınır (Bitiş Tarihi + Status otomatik).
- YP işlemlerde her işlemin kazancı işlem tarihindeki kur ile TL'ye çevrilir; "Maks. Kazanç – Toplam" kümülatif TL üzerinden kontrol edilir.
- Başlangıcı gelmemiş / Bitişi geçmiş kampanyalar günlük batch ile pasife alınır.
- Kampanya, ücret hesaplamasında standart tarifenin (2.5) üzerine uygulanır.

## 9. Statü Modeli

`status`: Active / Passive.

## 10. Veri Modeli

**`campaign`**: `id, campaign_code (unique), name, description, fixed_fee_gain_rate, commission_gain_rate, transaction_type, currency, start_date, end_date, min_tx_amount, max_gain_per_tx, max_gain_total, max_usage_count, status, ...`

**`customer_campaign`**: `id, customer_id, campaign_id, assigned_at, end_date, status` (müşteri-kampanya ilişkisi, tarihçeli)

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /campaigns` | GET | Liste |
| `POST /campaigns` | POST | Ekle |
| `PUT /campaigns/{id}` | PUT | Güncelle (kod hariç) |
| `POST /customers/{id}/campaign` | POST | Müşteriye kampanya ata (eski pasife) |

## 12. Entegrasyonlar

FX kuru (8.6) — YP kazanç TL karşılığı için.

## 13. Yetkilendirme ve Güvenlik

- Pazarlama/fiyatlandırma rolü.

## 14. Loglama ve Denetim İzi

- Kampanya ekleme/güncelleme ve müşteri-kampanya atama loglanır.

## 15. Hata Yönetimi

- Jenerik hata mesajı + kod.

## 16. Bildirimler

Tetiklenmez (opsiyonel: müşteriye kampanya bilgilendirmesi — kapsamda değil).

## 17. Batch / Zamanlanmış İşler

- Günlük: Başlangıcı gelmemiş / Bitişi geçmiş kampanyaların pasife alınması.

## 18. Bağımlılıklar

Müşteri Ücret ve Komisyonları (2.5), Müşteriler (2.1/2.2 — Kampanya combobox), Para Transferleri.

## 19. Açık Sorular ve Riskler

- "Maks. Kazanç – Toplam" ve "Maks. Kullanım Adedi" tüketim sayaçlarının nerede tutulacağı (customer_campaign üzerinde) netleştirilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Kampanya Kodu update'te değiştirilemez.
- [ ] Müşteriye ikinci kampanya atanınca mevcut otomatik pasife alınır.
- [ ] YP kazanç TL'ye çevrilerek toplam limit kontrol edilir.
- [ ] Süresi dolan kampanya batch ile pasifleşir.
