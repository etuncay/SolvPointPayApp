# 9.4 Yeni Doküman Tipi Tanımlama

> Ortak kurallar için bkz. `0.genel-standartlar.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Doküman Yönetim Sistemi › Yeni Doküman Tipi Tanımlama |
| Dosya | `9.4.yeni-dokuman-tipi-tanimlama.md` |
| Ekran tipi | Form (2 panel) |
| İlgili ENUM | document_category |

## 2. Amaç ve Kapsam

Yeni doküman tipinin tanımlandığı; saklama süreleri, boyut limiti ve erişim rollerinin belirlendiği ekrandır.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | Insert | Update |
|---|---|---|
| Sistem Yönetimi / Uyum | ✓ | ✓ |

## 4. Menü ve Navigasyon

Doküman Yönetim Sistemi menüsü altında (9.3'ten erişilir).

## 5. Ekran Yapısı ve Bileşenler

**Tanım alanları:** Doküman Kategorisi (`document_category`), Doküman Türü, Açıklama, Aktif Saklama Süresi (yıl), Arşivde Saklama Süresi (yıl), Max Dosya Boyutu, Kişisel Veri (checkbox), Onay Gerekir (checkbox).

**Erişim Paneli:** Görüntüleyebilen Roller (multiselect combobox), Onaylayabilen Roller (multiselect combobox).

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| Doküman Kategorisi / Türü | enum / string | ✓ | — |
| Açıklama | string | — | — |
| Aktif / Arşivde Saklama Süresi | int (yıl) | ✓ | — |
| Max Dosya Boyutu | number | — | Boşsa parametrik varsayılan |
| Kişisel Veri / Onay Gerekir | bool | — | — |
| Görüntüleyebilen Roller | enum çoklu | ✓ | Mevcut rol listesinden |
| Onaylayabilen Roller | enum çoklu | koşullu | Yalnızca "Onay Gerekir" seçiliyse aktif |

## 7. Validasyon Kuralları

- **Aynı Doküman Türü birden fazla kategoride olamaz.**
- Onaylayabilen Roller yalnızca "Onay Gerekir" seçiliyse aktif olur.
- `approval_required = true` ama onaylayabilen rol atanmadıysa **kaydetmez**.

## 8. İş Kuralları ve Akışlar

- `max_file_size_mb` boş bırakılırsa parametreden gelen varsayılan değer ile kısıtlanır.
- Roller mevcut rol listesinden seçilir.

## 9. Statü Modeli

Bu ekrana özgü statü modeli yoktur.

## 10. Veri Modeli

`dms.document_type` (bkz. 9.3) + **`dms.document_type_role`**: `id, document_type_id, role_id, access_type (View/Approve)`

|---|---|---|
| `POST /document-types` | POST | Yeni tip |
| `PUT /document-types/{id}` | PUT | Güncelle |

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Aynı doküman türü iki kategoride tanımlanamaz.
