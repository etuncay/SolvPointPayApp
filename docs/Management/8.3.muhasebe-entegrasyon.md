# 8.3 Muhasebe Entegrasyon

> Ortak kurallar için bkz. `0.genel-standartlar.md` (özellikle `0.10 Entegrasyon Statü Modeli`).

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Operasyonel Süreç Yönetimi › Muhasebe Entegrasyon |
| Dosya | `8.3.muhasebe-entegrasyon.md` |
| Ekran tipi | Listeleme + entegrasyon aksiyonları |
| İlgili ENUM | integration_status, transaction_type |

## 2. Amaç ve Kapsam

Gerçekleşen işlemlerin muhasebe sistemine gönderim durumunun izlendiği, hata alan kayıtların görüntülendiği ve yeniden gönderilebildiği ekrandır. Ortak **entegrasyon yaşam döngüsü** (`0.10`) işletilir.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | View (detay panel) | Tekrar Dene | İşlemi Durdur | İptal Et |
|---|---|---|---|---|---|
| Finans / Muhasebe | ✓ | ✓ | ✓ | ✓ | ✓ |

## 4. Menü ve Navigasyon

Operasyonel Süreç Yönetimi menüsü altında.

## 5. Ekran Yapısı ve Bileşenler

Tek tablo. **Kolonlar:** Referans No, İşlem Tarihi/Saati, İşlem Türü, Gönderen (Ad/Unvan + No), Alıcı (Ad/Unvan + No), Tutar + Para Birimi, Status (`integration_status`), Son Gönderim Tarihi, Servis Çıktısı, Deneme Sayısı.

Actions kolonunda Update/Delete yoktur; bunun yerine **Tekrar Dene / İşlemi Durdur / İptal Et** butonları (statüye göre). View ile ham JSON request/response detay panelinde gösterilir.

## 6. Alan Listesi

Tüm alanlar entegrasyon servisinden gelir. Detay paneli ham JSON mesajları içerir (yatay scroll engellemek için tabloda yer almaz).

## 7. Validasyon Kuralları

Salt-okunur izleme ekranı; aksiyonlar buton bazlıdır.

## 8. İş Kuralları ve Akışlar

- Statü akışı: `Pending → Preparing → Prepared → Sending → Sent → Completed`; hata dallanmaları `ErrorPrepare/ErrorSend/ErrorData`; `Retrying/OnHold/Canceled`.
- **Error\*** kayıtlar için: Tekrar Dene (Retrying), İşlemi Durdur (OnHold), İptal Et (Canceled).
- **OnHold** kayıtlar için: Tekrar Dene (Retrying), İptal Et (Canceled).
- Yeniden denemede önce dış sistemde kaydın oluşup oluşmadığı kontrol edilir (mükerrer engelleme — bkz. `0.10`).
- Varsayılan sıralama en yeniden en eskiye.

## 9. Statü Modeli

`integration_status` — bkz. `0.10`.

## 10. Veri Modeli

**`integration_record`** (muhasebe tipi): `id, integration_type=Accounting, reference_no, transaction_id, status, request_json, response_json, last_sent_at, attempt_count, ...` (ortak entegrasyon tablosu).

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /integrations/accounting` | GET | Liste |
| `GET /integrations/accounting/{id}` | GET | Detay + ham JSON |
| `POST /integrations/accounting/{id}/retry` | POST | Tekrar dene |
| `POST /integrations/accounting/{id}/hold` | POST | İşlemi durdur |
| `POST /integrations/accounting/{id}/cancel` | POST | İptal et |

## 12. Entegrasyonlar

Muhasebe sistemi (dış).

## 13. Yetkilendirme ve Güvenlik

- Finans / Muhasebe rolleri.

## 14. Loglama ve Denetim İzi

- Tekrar dene / durdur / iptal aksiyonları loglanır; correlation_id ile izlenir.

## 15. Hata Yönetimi

- Hata kayıtları statü + servis çıktısı ile gösterilir; ham JSON detayda.

## 16. Bildirimler

Tetiklenmez (kritik hatalar opsiyonel uyarı).

## 17. Batch / Zamanlanmış İşler

- Entegrasyon kuyruğunun işlenmesi ve otomatik retry politikası.

## 18. Bağımlılıklar

Para Transferleri (5), Finansal Mutabakat (8.5), Entegrasyonlar (12.7 — konfigürasyon).

## 19. Açık Sorular ve Riskler

- Yurt dışı/partner işlemlerin muhasebe hareketleri kaynak analizde detaylı tanımlanmamış (bkz. `0.19`); muhasebe gönderim payload'u bu boşluk netleşmeden tamamlanamaz.
