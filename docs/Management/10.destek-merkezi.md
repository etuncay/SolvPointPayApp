# 10. Destek Merkezi

> Ortak kurallar için bkz. `0.genel-standartlar.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Destek Merkezi |
| Dosya | `10.destek-merkezi.md` |
| Ekran tipi | Listeleme (grid) |
| Alt ekranlar | 10.1 Yeni Talep Giriş, 10.2 Raporlama |
| İlgili ENUM | complaint_type, level, case_status |

## 2. Amaç ve Kapsam

Tüm talep ve şikâyetlerin listelendiği ekrandır. Talep girişi/yönetimi 10.1'de, raporlama 10.2'de yapılır.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | View | Insert | Update |
|---|---|---|---|---|
| Destek personeli | ✓ | ✓ | ✓ | ✓ |
| İlgili departman personeli | atanmışlar | ✓ | — | ✓ |

## 4. Menü ve Navigasyon

Sol menüde "Destek Merkezi"; altında 10.1, 10.2. **Tarafıma Atanmış Talepler** butonu.

## 5. Ekran Yapısı ve Bileşenler

Tek panelli grid. **Kolonlar:** Talep No, Konu, Talep Tipi (`complaint_type`), Sorumlusu, Aciliyeti (`level`), Kritikliği (`level`), Oluşturma Tarihi, Son Güncelleme Tarihi, Talep Yaşı, Status (`case_status`).

## 6. Alan Listesi

| Kolon | Tip | Aranabilir | Açıklama |
|---|---|---|---|
| Talep No | string | ✓ | — |
| Konu | string | ✓ | — |
| Talep Tipi | enum | ✓ | complaint_type |
| Sorumlusu | user | ✓ | — |
| Aciliyeti / Kritikliği | enum | ✓ | level |
| Oluşturma / Son Güncelleme Tarihi | datetime | ✓ (aralık) | — |
| Talep Yaşı | süre | — | Hesaplanmış |
| Status | enum | ✓ | case_status |

## 7. Validasyon Kuralları

Listeleme ekranıdır; talep validasyonu 10.1'de.

## 8. İş Kuralları ve Akışlar

- Satıra tıklayınca 10.1 Yeni Talep Giriş ekranı View/Update modunda açılır.
- "Tarafıma Atanmış Talepler" butonu kullanıcının sorumlu olduğu talepleri filtreler.
- Banka Mutabakatı (6.4) Unmatched satırları otomatik talep açar; bu talepler de burada listelenir.

## 9. Statü Modeli

`case_status` — bkz. `0.18` (Unassigned, Assigned, InProgress, WaitingFor*, Escalated, ReOpened, Resolved_*, Rejected).

## 10. Veri Modeli

**`case`** (talep): `case_id, subject, complaint_type, owner_user_id, urgency, criticality, case_status, requester_type, requester_id, department_id, created_at, updated_at, ...`

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /support-cases` | GET | Filtreli liste |
| `GET /support-cases/{id}` | GET | Detay (10.1) |

## 12. Entegrasyonlar

Yoktur (otomatik talep kaynağı: Banka Mutabakatı 6.4).

## 13. Yetkilendirme ve Güvenlik

- Rol bazlı; atanmış talep görünürlüğü.

## 14. Loglama ve Denetim İzi

- Talep listeleme/erişim loglanır.

## 15. Hata Yönetimi

- Jenerik hata mesajı + kod.

## 16. Bildirimler

Tetiklenmez (bildirimler 10.1 aksiyonlarında).

## 17. Batch / Zamanlanmış İşler

Yoktur.

## 18. Bağımlılıklar

Yeni Talep Giriş (10.1), Raporlama (10.2), Banka Mutabakatı (6.4).

## 19. Açık Sorular ve Riskler

- Talep Yaşı'nın açık/kapalı taleplerde nasıl hesaplanacağı (kapanışta donar mı) netleştirilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları
