# 5.2 İade / İptal / Düzeltme

> Ortak kurallar için bkz. `0.genel-standartlar.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Para Transferleri › İade / İptal / Düzeltme |
| Dosya | `5.2.iade-iptal-duzeltme.md` |
| Ekran tipi | Form (manuel para transferi) — onay zorunlu |
| İlgili ENUM | currency, correction_reason |

## 2. Amaç ve Kapsam

Operasyonel hata, iade veya iptal nedeniyle **bir cüzdandan başka bir cüzdana manuel para transferi** yapılan ekrandır. Onay zorunlu olduğundan kayıt doğrudan kaydedilmez, **Onaya Gönder** ile onay akışına girer.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | Insert | Onaya Gönder | Onayla |
|---|---|---|---|
| Yetkili Operasyon | ✓ | ✓ | — |
| Onaycı (1./2.) | — | — | ✓ |

Manuel transfer yalnızca yetkili roller tarafından yapılır; yüksek riskli tutarlarda **maker-checker (ikinci onay)** zorunludur.

## 4. Menü ve Navigasyon

Para Transferleri menüsü altında. **Butonlar:** Vazgeç, Onaya Gönder. Onaya Gönder tetiklenince kullanıcı İşlem Detay (5.1) ekranına yönlendirilir.

## 5. Ekran Yapısı ve Bileşenler

Tek formlu ekran. **Alanlar:** Talep/Şikâyet No, İşlem No (iade/iptal/düzeltmeye sebep olan işlem — opsiyonel), Kaynak Müşteri, Kaynak Cüzdan, Hedef Müşteri, Hedef Cüzdan, Aktarılacak Tutar, Para Birimi, İşlem Açıklama, Aktarım Sebebi (`correction_reason`), Manüel İşlem Açıklaması. Opsiyonel ek doküman iliştirilebilir.

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| Talep/Şikâyet No | string | — | İlişkili Destek Merkezi talebi |
| İşlem No | string | — | Sebep olan işlem |
| Kaynak / Hedef Müşteri | string | ✓ | — |
| Kaynak / Hedef Cüzdan | string | ✓ | Herhangi bir cüzdan olabilir |
| Aktarılacak Tutar | number | ✓ | Talep para birimi cinsinden |
| Para Birimi | enum | ✓ | Talep para birimi |
| İşlem Açıklama | string | — | — |
| Aktarım Sebebi | enum | ✓ | correction_reason |
| Manüel İşlem Açıklaması | string | ✓ | — |
| Ek Doküman | dosya | — | İşleme bağlanır, sonradan değiştirilemez |

## 7. Validasyon Kuralları

- Aktarım Sebebi + Manüel İşlem Açıklaması **zorunludur**.
- Kaynak hesap **Kasa (Reserve)** ise işlem tutarı parametrik **maksimum tutarı** aşamaz.

## 8. İş Kuralları ve Akışlar

- Girilen Aktarılacak Tutar + Talep Para Birimi esas alınır. Sistem bu tutarı kullanılacak kurla:
  - Kaynak cüzdanın para birimine çevirerek **Kaynak Cüzdandan Çıkış Tutarı**,
  - Hedef cüzdanın para birimine çevirerek **Hedef Cüzdana Giriş Tutarı** hesaplar; her ikisi de para biriminin minor unit kuralına göre yuvarlanır.
- Bu ekrandan **herhangi bir cüzdandan herhangi bir cüzdana** transfer yapılabilir (düzeltme amaçlı).
- Onay zorunlu olduğundan buton "Onaya Gönder"dir; tetiklenince kullanıcı İşlem Detay (5.1) ekranına yönlendirilir. Kullanıcı Geri Dön ile forma dönüp düzenleyebilir veya Onaya Gönder ile talebi onay akışına iletir.
- İliştirilen dokümanlar işleme bağlanır ve sonradan değiştirilemez.

## 9. Statü Modeli

Oluşan işlem `transaction` olarak yönetilir (bkz. `0.9`); onay süreci Onay Havuzu (8.1) statü modeline tabidir.

## 10. Veri Modeli

`transaction` (manuel transfer tipiyle) + `correction_request` (`id, complaint_id, source_transaction_id, source_wallet_id, target_wallet_id, requested_amount, requested_currency, source_out_amount, target_in_amount, correction_reason, manual_description, created_by`).

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `POST /corrections` | POST | Manuel transfer talebi oluştur (onaya gönder) |
| `GET /fx/rate` | GET | Çevrim kuru |
| `POST /documents` | POST | Ek doküman yükleme |

## 12. Entegrasyonlar

Kur/FX (8.6), Doküman Yönetim Sistemi (9), Onay Havuzu (8.1).

## 13. Yetkilendirme ve Güvenlik

- Yalnızca yetkili roller; yüksek tutarda 2. onay (maker-checker) zorunlu (Onay Kuralları 12.2).

## 14. Loglama ve Denetim İzi

- Manuel transfer talepleri (kaynak/hedef/tutar/sebep) detaylı loglanır — yüksek riskli işlem.

## 15. Hata Yönetimi

- Kasa kaynaklı işlemde parametrik max tutar aşımında anlamlı hata.
- Jenerik hata mesajı + kod.

## 16. Bildirimler

- Onay sonucuna göre talep sahibine/taraflara bilgilendirme (parametrik).

## 17. Batch / Zamanlanmış İşler

Yoktur.

## 18. Bağımlılıklar

İşlem Detay (5.1), Onay Havuzu (8.1), Kur/FX (8.6), Destek Merkezi (10), DMS (9).

## 19. Açık Sorular ve Riskler

- "Kullanılacak kur"un kaynağı (anlık FX / işlem kuru) netleştirilmeli.
- Hangi tutarın 2. onay gerektirdiği Onay Kuralları (12.2) ile tanımlanmalı.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Aktarım Sebebi + Manüel Açıklama girilmeden Onaya Gönder pasiftir.
- [ ] Kasa kaynaklı işlemde parametrik max tutar aşılamaz.
- [ ] Kaynak çıkış / hedef giriş tutarları doğru kur ve yuvarlama ile hesaplanır.
- [ ] Onaya Gönder kullanıcıyı İşlem Detay'a yönlendirir.
- [ ] İliştirilen doküman sonradan değiştirilemez.
