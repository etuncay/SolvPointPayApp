# 3.3.1 Temsilci-Grup İlişkisi

> Ortak kurallar için bkz. `0.genel-standartlar.md`. Üst ekran: `3.3.temsilci-grup-yonetimi.md`.

| Alan | Değer |
|---|---|
| Menü | Temsilciler › Temsilci Grup Yönetimi › Temsilci-Grup İlişkisi |
| Dosya | `3.3.1.temsilci-grup-iliskisi.md` |
| Ekran tipi | Listeleme + satır içi atama (menüde yer almaz) |

## 2. Amaç ve Kapsam

Seçili temsilci grubuna ait temsilcilerin listelendiği ve temsilcilerin ilgili gruba **manuel atandığı** ekrandır. Grup bilgileri değiştirilmez; yalnızca temsilci-grup ilişkisi yönetilir.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | Insert (atama) | Update | Delete (çıkarma) |
|---|---|---|---|---|
| Genel Müdürlük Operasyon | ✓ | ✓ | ✓ | ✓ |
| Finans / Fiyatlandırma | ✓ | ✓ | ✓ | ✓ |

## 4. Menü ve Navigasyon

Menüde **listelenmez**; yalnızca Temsilci Grup Yönetimi (3.3) ekranından gereken hallerde açılır. Üstte seçili grubun Grup Kodu ve Grup Adı salt-okunur gösterilir.

## 5. Ekran Yapısı ve Bileşenler

Tek tablo; satır içi atama. **Kolonlar:** Temsilci Kodu, Temsilci Adı, İl, İlçe, İşlem Başı Ortalama Ücret, Ortalama İşlem Adedi, Mevcut Grup, Grup Atama Tarihi, Status (`status`). Varsayılan görünüm yalnızca aktif atamalar; pasif atamalar filtre ile gelir.

## 6. Alan Listesi

| Alan | Tip | Düzenlenebilir | Açıklama |
|---|---|---|---|
| Temsilci Kodu / Adı | string | atamada seçilir | — |
| İl / İlçe | string | hayır | Yalnızca filtreleme/seçim kolaylığı |
| İşlem Başı Ortalama Ücret / Ortalama İşlem Adedi | number | hayır | Hesaplanmış (son 1 ay, tamamlanmış işlemler) |
| Mevcut Grup | string | sistem | Her zaman dolu olmalı |
| Grup Atama Tarihi | date | sistem | Atama kaydının başlangıç tarihi |
| Status | enum | sistem | Active/Passive |

## 7. Validasyon Kuralları

- Pasif gruplara yeni atama yapılamaz (yalnızca mevcut atamalar görüntülenir).

## 8. İş Kuralları ve Akışlar

- Bir temsilci **aynı anda yalnızca bir aktif gruba** bağlı olabilir. Yeni atamada önceki aktif atama otomatik pasife alınır, yeni atama kaydı oluşur.
- Atamalar **tarihsel olarak izlenebilir**; yeniden atamada yeni kayıt ve yeni Grup Atama Tarihi.
- Gruptan çıkarma = ilişkinin pasife alınması (temsilci silinmez). Aktif grubu kalmayan temsilci **varsayılan gruba** atanır.
- Mevcut Grup kolonu her zaman dolu olmalıdır.

## 9. Statü Modeli

`status`: Active / Passive (atama kaydı bazında).

## 10. Veri Modeli

**`agent_group_assignment`**: `id, agent_id, group_code, assigned_at, status, created_at/by, record_status`

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /agent-groups/{code}/agents` | GET | Gruba bağlı temsilciler |
| `POST /agent-groups/{code}/agents` | POST | Temsilci ata (eski atama pasife) |
| `DELETE /agent-group-assignments/{id}` | DELETE | Gruptan çıkar (varsayılana ata) |
