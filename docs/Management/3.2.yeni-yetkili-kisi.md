# 3.2 Yeni Yetkili Kişi

> Ortak kurallar için bkz. `0.genel-standartlar.md`. Insert/Update/View ekranı aynıdır.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Temsilciler › Yeni Yetkili Kişi |
| Dosya | `3.2.yeni-yetkili-kisi.md` |
| Ekran tipi | Tek sayfa form — Insert/Update/View |
| İlgili ENUM | identity_document, kyc_level, entity_status, marital_status, gender, education_level, employment_category, employment_occupation, visa_type, address_type |

## 2. Amaç ve Kapsam

Temsilci adına işlem yapabilen veya sisteme erişen **yetkili kişi** (gerçek kişi) kaydının oluşturulduğu ekrandır. Yetkili kişi; tüzel müşteri/temsilci adına hareket eden, aday veya bireysel müşteri kaydı da olabilen gerçek kişidir.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | Insert | Update | View | Bloke Et/Kaldır |
|---|---|---|---|---|
| Genel Müdürlük Operasyon | ✓ | ✓ | ✓ | yetkiye bağlı |
| Uyum | — | ✓ | ✓ | ✓ |

## 4. Menü ve Navigasyon

Temsilciler menüsü altında; ayrıca Temsilci/Tüzel Müşteri ekranlarındaki Yetkililer panelinden kişi tanımına geçilebilir.

## 5. Ekran Yapısı ve Bileşenler

**Butonlar:** Kişi Aktiviteleri, Bloke Et, Bloke Kaldır, Belge Yükleme, Belge Görme/İndirme, Taslak Kaydet, Kaydet, Vazgeç.

**Üst alanlar:** Adı ve Soyadı, Uyruğu, Kimlik No, Kimlik Tipi, Doğum Tarihi.

**Paneller:**

1. **Kişisel Bilgiler Paneli** — Kişi No (değiştirilemez), KYC Seviyesi, Oluşturma Tarihi, Status (+gerekçe). *Yeni kayıtta gösterilmez.*
2. **Erişim Bilgileri** — Son Login, Başarısız Erişim Teşebbüsü Sayısı, Cihaz Bilgileri, IP Lokasyonu (salt-okunur). *Yeni kayıtta gösterilmez.*
3. **Nüfus Bilgileri Paneli** — Doğum Yeri, Medeni Durum, Seri No/Doküman No, Veriliş Tarihi, Veren Makam, Geçerlilik Tarihi, Anne/Baba Adı, Cinsiyet.
4. **Detay Bilgiler Paneli** — Evlenmeden Önceki Soyadı, Vergi Ülkesi, Eğitim Durumu, Çalışma Durumu, Mesleği.
5. **Yabancı Bireysel Kişi Paneli** — Vize Tipi, Vize Bitiş Tarihi, Oturum İzni, Oturum İzni Bitiş Tarihi, Doğum Ülkesi, Yerleşik Olduğu Ülke. Yalnızca yabancı kişilerde.
6. **Adres Paneli** — çoklu adres; yabancılarda Ülke/İl/İlçe/Posta Kodu/Adres (textarea).
7. **İletişim Bilgileri Paneli** — E-posta/Telefon; doğrulama; asıl kanal.

## 6. Alan Listesi

Türk vatandaşlarında doğum tarihi girilince Nüfus Bilgileri **KPS'ten dolar ve değiştirilemez**. Kimlik No girilince mevcut kişi varsa ekran o bilgilerle dolar.

## 7. Validasyon Kuralları

| Alan | Kontrol |
|---|---|
| Adı ve Soyadı | Boş olamaz; 2–255 karakter; sayısal/özel karakter yok |
| Uyruğu / Kimlik Tipi / Medeni Durum / Cinsiyet / Vergi Ülkesi / Eğitim / Çalışma / Meslek | Seçim zorunlu |
| Kimlik No | TUR: 11 hane, rakam, ilk hane ≠ 0; Yabancı: Pasaport 1–20 alfanümerik |
| Doğum Tarihi | GG.AA.YYYY; gelecek tarih olamaz; **18 yaş altı olamaz** |
| Doğum Yeri | 2–100 karakter |
| Seri No/Doküman No | 5–30 karakter |
| Veriliş / Geçerlilik Tarihi | GG.AA.YYYY; mantık kontrolleri (2.1 ile aynı) |
| Anne/Baba Adı | Kimlik Ülkesi TUR ise zorunlu; 2–50 karakter |
| Evlenmeden Önceki Soyadı | Kadın + Married ise zorunlu |
| Vize alanları | Uyruk ≠ TUR ise zorunlu |
| Adres / İletişim | bkz. 2.1 kuralları |

## 8. İş Kuralları ve Akışlar

- **Kaydet akışı:** zorunlu alan → belge kontrolü (asgari `Identity` kategorisinde `false`/`Approved` belge) → arka planda kimlik doğrulama (OCR↔KPS) + sanction/kara liste.
- Mevcut kişi güncellemede Nüfus Bilgileri değişmişse Kaydet'te kimlik doğrulama + sanction tekrar çalışır. KYC seviyesi yükseltme, adres/ülke değişikliği sanction "temiz" sonucu olmadan tamamlanamaz; kara liste kaydı bulunursa Status=Blocked.
- Taslak kayıt → Inactive.
- **18 yaş altı yetkili kişi kabul edilmez** (bireysel müşteriden farkı: bireysel müşteri kaydedilebilir ama işlem yapamaz — bkz. `0.19`).

## 9. Statü Modeli

`entity_status`: Inactive → Active → Blocked → Active/Closed. KYC akışı `0.7`.

## 10. Veri Modeli

- **`person`** (3.2 kişi için ortak tablo).
- **`authorized_person`** ilişki tabloları: `agent_authorized_person` / `corporate_authorized_person` (yetkilendirme bağlamı burada değil, ilgili tüzel/temsilci ekranında).
- **`person_address`**, **`person_contact`**, **`person_foreign_info`**.

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /authorized-persons/{id}` | GET | Detay |
| `POST /authorized-persons` | POST | Yeni kayıt |
| `PUT /authorized-persons/{id}` | PUT | Güncelle |
| `POST /authorized-persons/draft` | POST | Taslak |
| `POST /authorized-persons/{id}/block` / `unblock` | POST | Bloke |

## 12. Entegrasyonlar

KPS (kimlik doğrulama), Sanction screening, Doküman Yönetim Sistemi.

## 13. Yetkilendirme ve Güvenlik

- Bloke Et/Kaldır yetki bazlı, loglanır.
- PII ağırlıklı ekran; erişim loglanır.

## 14. Loglama ve Denetim İzi

- Insert/Update/Bloke eski-yeni değer ile loglanır.

## 15. Hata Yönetimi

- KPS/sanction erişilemezse arka plan kontrolü beklemede.

## 16. Bildirimler

- İletişim değişiminde eski adrese bilgilendirme; telefon OTP / e-posta doğrulama.

## 17. Batch / Zamanlanmış İşler

- Aylık periyodik sanction taraması kapsamında (temsilci/tüzel yetkilileri).

## 18. Bağımlılıklar

`person`, Temsilci (3.1), Tüzel Müşteri (2.2), DMS (9), KYC Yönetimi (8.2).

## 19. Açık Sorular ve Riskler

- Yetkili kişi statüsü Inactive yapıldığında bağlı olduğu tüm temsilci/tüzel yetkilendirmelerinin davranışı netleştirilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] 18 yaş altı yetkili kişi reddedilir.
- [ ] Türk vatandaşında Nüfus Bilgileri KPS'ten dolar ve salt-okunurdur.
- [ ] Kaydet'te belge + kimlik doğrulama + sanction tetiklenir.
- [ ] Nüfus Bilgileri değişimi sanction taramasını yeniden tetikler.
- [ ] Taslak kayıt Inactive statüde oluşur.
