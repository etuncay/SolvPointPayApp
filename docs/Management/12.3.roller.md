# 12.3 Roller

> Ortak kurallar için bkz. `0.genel-standartlar.md` (özellikle `0.5`).

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Sistem Yönetimi › Roller |
| Dosya | `12.3.roller.md` |
| Ekran tipi | Listeleme + detay (tablı) |
| İlgili ENUM | status |

## 2. Amaç ve Kapsam

Rollerin tanımlandığı; ekran yetkileri ve alan limit/onay yetkilerinin belirlendiği ekrandır.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | Insert | Update |
|---|---|---|---|
| Sistem Yöneticisi | ✓ | ✓ | ✓ |

## 4. Menü ve Navigasyon

Sistem Yönetimi menüsü altında. Detay ekranı **Ekran Yetkileri** ve **Limitler & Onaylar** tablarından oluşur.

## 5. Ekran Yapısı ve Bileşenler

**Liste kolonları:** Rol Adı, Açıklama, Status (`status`), Oluşturma Tarihi, Son Güncelleme Tarihi.

**Ekran Yetkileri Tabı:** Tüm ekranların listelendiği, hücreleri checkbox olan matris — Ekran, List, View/Details, Insert, Update, Delete, Export, 1. Onaycı Olabilir, 2. Onaycı Olabilir (hepsi BOOL).

**Limitler & Onaylar Tabı:** İşlem Adı (combobox), Ekran Adı (combobox), 1. Onaycı Olabilir (BOOL), 2. Onaycı Olabilir (BOOL). Satır eklenebilir/güncellenebilir.

## 6. Alan Listesi

| Alan | Tip | Açıklama |
|---|---|---|
| Rol Adı / Açıklama | string | — |
| Status | enum | Active/Passive |
| Ekran yetkileri | bool matris | List/View/Insert/Update/Delete/Export/1.Onaycı/2.Onaycı |
| Limitler & Onaylar | tablo | İşlem/ekran bazlı onaycı yetkileri |

## 7. Validasyon Kuralları

- Rol Adı tekildir.

## 8. İş Kuralları ve Akışlar

- **1. Onaycı Olabilir**: başka kullanıcının girdiği işlemde 1. onaycı olma yetkisi. **2. Onaycı Olabilir** benzer.
- Pasif rol yeni kullanıcıya atanamaz (bkz. 12.1).
- Ekran gizleme tek başına yetki değildir; tüm işlemler yetki matrisinden geçer (bkz. `0.5`).

## 9. Statü Modeli

`status`: Active / Passive.

## 10. Veri Modeli

**`role`**: `id, name (unique), description, status, created_at, updated_at`
**`role_screen_permission`**: `id, role_id, screen, can_list, can_view, can_insert, can_update, can_delete, can_export, can_first_approve, can_second_approve`
**`role_field_approval`**: `id, role_id, operation_name, screen, can_first_approve, can_second_approve`

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /roles` | GET | Liste |
| `GET /roles/{id}` | GET | Detay (yetki matrisi) |
| `POST /roles` | POST | Yeni rol |
| `PUT /roles/{id}` | PUT | Güncelle |
