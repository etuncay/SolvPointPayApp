## 2. Amaç ve Kapsam

**Temsilci grupları bazında** ücret ve komisyon tanımlarının yapıldığı ekrandır. Tanımlar, ilgili gruba bağlı temsilcilerin işlemlerinde uygulanır. Değişiklikler **ileriye dönük** geçerlidir.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | Insert | Update | Export |
|---|---|---|---|---|
| Finans / Fiyatlandırma | ✓ | ✓ | ✓ | ✓ |
| Operasyon | ✓ | — | — | ✓ |

## 4. Menü ve Navigasyon

Temsilciler menüsü altında. Yeni ekleme **modal** ile; detay sayfası yoktur.

## 5. Ekran Yapısı ve Bileşenler

Tek tablo. **Kolonlar:** Grup Kodu, İşlem (`transaction_type`), Para Birimi (`currency`), Alt Limit, Sabit Ücret, Değişken Ücret (%), Başlangıç Tarihi, Bitiş Tarihi, Status (`status`). Varsayılan görünüm yalnızca aktif kayıtlar; pasifler filtre ile gelir.

Yeni ekleme modalında Grup Kodu mevcut grup listesinden seçilir.

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| Grup Kodu | enum | ✓ | Mevcut grup listesinden |
| İşlem / Para Birimi | enum | ✓ | — |
| Alt Limit | number | ✓ | ≥ 0; skala alt eşiği |
| Sabit Ücret / Değişken Ücret (%) | number | ✓ | — |
| Başlangıç / Bitiş Tarihi | date | — | Boş başlangıç = kayıt tarihi |
| Status | enum | sistem | Active/Passive |

## 7. Validasyon Kuralları

- Alt Limit zorunlu ve ≥ 0.
- Her Grup Kodu + İşlem + Para Birimi kombinasyonu için **Alt Limit = 0 olan en az bir aktif kayıt** zorunludur; aksi halde ücret hesaplanmaz.
- Pasif gruplara yeni ücret/komisyon tanımı yapılamaz.

## 8. İş Kuralları ve Akışlar

- Aynı Grup Kodu + İşlem + Para Birimi + Alt Limit kombinasyonu için aynı anda tek aktif kayıt. Yeni kayıtta uyarı + validasyon, mevcut aktif kayıt pasife alınır (Bitiş Tarihi + Status otomatik).
- Farklı Alt Limit değeri **yeni kademe** olarak eklenir; mevcut kademeler korunur.
- **Hesaplama:** işlem anında temsilcinin bağlı olduğu **aktif grup** üzerinden; işlem tutarına ≤ en yüksek Alt Limit kaydı seçilir. `Temsilciye Ödenecek Ücret = Sabit Ücret + Değişken Ücret(%) × İşlem Tutarı`. Para biriminin minor unit kuralına göre yuvarlama.
- Başlangıcı gelmemiş / Bitişi geçmiş kayıtlar günlük batch ile pasife alınır.

## 9. Statü Modeli

`status`: Active / Passive. Tarihçeli.

## 10. Veri Modeli

**`agent_fee`**: `id, group_code, transaction_type, currency, lower_limit, fixed_fee, variable_fee_pct, start_date, end_date, status, created_at/by, updated_at/by, record_status`

## 12. Entegrasyonlar

FX kuru (8.6) — gerekirse TL karşılığı.

## 13. Yetkilendirme ve Güvenlik

- Fiyatlandırma rolü; değişiklik Onay Havuzu'na tabi olabilir (12.2).

## 14. Loglama ve Denetim İzi

- Ücret/komisyon ekleme/güncelleme eski-yeni değer ile loglanır.

## 15. Hata Yönetimi

- Alt Limit = 0 kaydı yoksa hesaplama servisi anlamlı hata kodu döner.

## 16. Bildirimler

Tetiklenmez.

## 17. Batch / Zamanlanmış İşler

- Günlük: Başlangıcı gelmemiş / Bitişi geçmiş ücret kayıtlarının pasife alınması.

## 18. Bağımlılıklar

Temsilci Grup Yönetimi (3.3), Para Transferleri (komisyon tüketicisi), Kur/FX (8.6).

## 19. Açık Sorular ve Riskler

- Temsilci komisyonunun gönderen/alıcı temsilci payı (Cs/Cr) dağılımının bu tanımla ilişkisi netleştirilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Alt Limit = 0 kaydı olmadan ücret hesaplanamaz.
- [ ] Kademe seçimi "≤ tutar en yüksek Alt Limit" mantığıyla çalışır.
- [ ] Yeni kayıt çakışan aktif kaydı pasife alır.
- [ ] Pasif gruba ücret tanımı yapılamaz.
- [ ] Süresi dolan kayıtlar batch ile pasifleşir.
