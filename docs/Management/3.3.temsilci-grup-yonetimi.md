# 3.3 Temsilci Grup Yönetimi

> Ortak kurallar için bkz. `0.genel-standartlar.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Temsilciler › Temsilci Grup Yönetimi |
| Dosya | `3.3.temsilci-grup-yonetimi.md` |
| Ekran tipi | Listeleme + satır içi ekleme/güncelleme |
| Detay/alt ekran | `3.3.1.temsilci-grup-iliskisi.md` |
| İlgili ENUM | status |

## 2. Amaç ve Kapsam

Temsilci gruplarının (ücret/komisyon hesaplamasının temel sınıflandırması) tanımlandığı ana ekrandır. Bu ekranda yalnızca **grup master verisi** yönetilir; ücret/komisyon tanımı 3.4'te yapılır.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | Insert | Update | Delete |
|---|---|---|---|---|
| Genel Müdürlük Operasyon | ✓ | ✓ | ✓ | (pasife alma) |
| Finans / Fiyatlandırma | ✓ | ✓ | ✓ | (pasife alma) |

## 4. Menü ve Navigasyon

Temsilciler menüsü altında. **Temsilci-Grup İlişkisi** (3.3.1) menüde yer almaz; yalnızca gereken hallerde bu ekrandan açılır.

## 5. Ekran Yapısı ve Bileşenler

Tek tablo; satır içi ekleme/güncelleme. **Kolonlar:** Grup Kodu, Grup Adı, Açıklama, Temsilci Sayısı, İşlem Başı Ortalama Ücret, Temsilci Başı Ortalama İşlem Adedi, Status (`status`).

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| Grup Kodu | string | ✓ | Sistemsel anahtar; update'te değiştirilemez |
| Grup Adı / Açıklama | string | ✓ / — | — |
| Temsilci Sayısı | int | sistem | Hesaplanmış; aktif temsilci sayısı |
| İşlem Başı Ortalama Ücret | number | sistem | Hesaplanmış; son 1 ay; TL; günde 1 kez batch |
| Temsilci Başı Ortalama İşlem Adedi | number | sistem | Hesaplanmış; son 1 ay; günde 1 kez batch |
| Status | enum | sistem | Active/Passive |

## 7. Validasyon Kuralları

- Grup Kodu tekildir; update'te değiştirilemez.
- Grup Adı boş olamaz.

## 8. İş Kuralları ve Akışlar

- Hesaplanmış alanlar (Temsilci Sayısı, ortalama ücret/adet) salt-okunurdur; **günde 1 kez batch** ile hesaplanır, yalnızca tamamlanmış işlemler dikkate alınır, son 1 ayda işlemi olmayan gruplarda "-" gösterilir. Ortalama ücret işlem tarihindeki kurla TL üzerinden hesaplanır.
- Bir grup pasife alınmadan önce **bağlı aktif temsilci** kontrolü yapılır; aktif temsilcisi olan grup pasife alınamaz, kullanıcı bilgilendirilir.
- Pasife alma geçmiş atamaları/ücret tanımlarını silmez; yalnızca ileriye dönük kullanımı engeller.
- Sistemde **en az bir varsayılan temsilci grubu** zorunludur; varsayılan grup silinemez/pasife alınamaz. Yeni temsilci otomatik varsayılan gruba atanır.

## 9. Statü Modeli

`status`: Active / Passive.

## 10. Veri Modeli

**`agent_group`**: `id, group_code (unique), name, description, is_default, status, created_at/by, updated_at/by, record_status`

Hesaplanmış metrikler için: **`agent_group_metric`**: `group_code, calc_date, agent_count, avg_fee_per_tx, avg_tx_count_per_agent`

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /agent-groups` | GET | Liste (metriklerle) |
| `POST /agent-groups` | POST | Ekle |
| `PUT /agent-groups/{id}` | PUT | Güncelle (kod hariç) |
| `POST /agent-groups/{id}/deactivate` | POST | Pasife al (aktif temsilci kontrolü) |

## 12. Entegrasyonlar

FX kuru (8.6) — ortalama ücret TL hesabı için.

## 13. Yetkilendirme ve Güvenlik

- Rol bazlı; varsayılan grup koruması sistem kuralıdır.

## 14. Loglama ve Denetim İzi

- Grup ekleme/güncelleme/pasife alma loglanır.

## 15. Hata Yönetimi

- Aktif temsilcisi olan grup pasife alınmaya çalışılırsa anlamlı hata + bilgilendirme.

## 16. Bildirimler

Tetiklenmez.

## 17. Batch / Zamanlanmış İşler

- Günlük: İşlem Başı Ortalama Ücret ve Temsilci Başı Ortalama İşlem Adedi hesaplanması.

## 18. Bağımlılıklar

Temsilci (3.1), Temsilci-Grup İlişkisi (3.3.1), Temsilci Ücret ve Komisyonları (3.4).

## 19. Açık Sorular ve Riskler

- **Tutarsızlık:** Ortalama metrikler bu ekranda "günde 1 kez", Temsilci-Grup İlişkisi (3.3.1) ekranında "ayda 1 kez" deniyor. Tek frekans (günlük öneriliyor) standardize edilmeli (bkz. `0.19`).

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Grup Kodu update'te değiştirilemez.
- [ ] Aktif temsilcisi olan grup pasife alınamaz.
- [ ] Varsayılan grup silinemez/pasife alınamaz.
- [ ] Hesaplanmış metrikler günlük batch ile dolar; işlemi olmayan grupta "-" gösterilir.
