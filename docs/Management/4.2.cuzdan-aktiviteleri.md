| Alan | Değer |

| Menü | Dijital Cüzdanlar › Cüzdan Detay › Cüzdan Aktiviteleri |

## 2. Amaç ve Kapsam

## 4. Menü ve Navigasyon

`4.1` Cüzdan Detay'dan "Cüzdan Aktiviteleri" butonu ile açılır. Satıra tıklayınca İşlem Detay (5.1) açılır.

## 5. Ekran Yapısı ve Bileşenler

Tek tablo. **Kolonlar:** İşlem Numarası, İşlem Tarihi, İşlem Yönü (`transaction_direction`), Karşı Taraf No, Karşı Taraf Adı ve Soyadı / Unvanı, Karşı Hesap (Cüzdan No / IBAN), Gönderen Temsilci No, Alıcı Temsilci No, Referans No, İşlem Türü (`transaction_type`), Para Birimi, Tutar, İşlem Sonrası Bakiye, Status (`transaction_status`).

## 6. Alan Listesi

| Kolon | Tip | Açıklama |
|---|---|---|
| İşlem Numarası / Tarihi | string / datetime | — |
| İşlem Yönü | enum | Bakiye etkisine göre: artıyorsa Para Girişi, azalıyorsa Para Çıkışı |
| Karşı Taraf No / Adı | string | — |
| Karşı Hesap | string | Cüzdan No veya IBAN |
| Gönderen / Alıcı Temsilci No | string | — |
| Referans No / İşlem Türü / Para Birimi | — | — |
| Tutar | number | Her zaman pozitif; yön İşlem Yönü ile |
| İşlem Sonrası Bakiye | number | Bu cüzdan perspektifi |
| Status | enum | transaction_status |

## 7. Validasyon Kuralları

Salt-okunur listeleme ekranıdır.

## 8. İş Kuralları ve Akışlar

- Yalnızca seçili cüzdana giren/çıkan işlemler gösterilir.
- **İşlem Yönü** işlem tipine göre değil **bakiyeye etkisine** göre belirlenir.
- Tutar her zaman pozitif gösterilir; artı/eksi etkisi İşlem Yönü ile ifade edilir.
- İşlem Sonrası Bakiye, gönderen/alıcı ayrımı yapılmaksızın tamamen seçili cüzdan perspektifidir.
- Varsayılan sıralama İşlem Tarihi'ne göre yeniden eskiye; kullanıcı kolon bazlı sıralayabilir.

## 9. Statü Modeli

`transaction_status` — bkz. `0.9`.

## 10. Veri Modeli

Kaynak `transaction` + `transaction_account_movement` (cüzdan bazlı hareket satırları). Bu ekran filtreli okuma yapar.

**`transaction_account_movement`**: `id, transaction_id, wallet_id, direction, amount, post_balance, currency, created_at`
