# 8.4 BTRANS Raporları

> Ortak kurallar için bkz. `0.genel-standartlar.md` (özellikle `0.10`).

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Operasyonel Süreç Yönetimi › BTRANS Raporları |
| Dosya | `8.4.btrans-raporlari.md` |
| Ekran tipi | Listeleme + entegrasyon aksiyonları |
| İlgili ENUM | integration_status |

## 2. Amaç ve Kapsam

BTRANS raporlarının gönderim durumunun izlendiği, hata alan kayıtların görüntülendiği ve yeniden gönderilebildiği ekrandır. Ortak entegrasyon yaşam döngüsü işletilir.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | View | Tekrar Dene | İşlemi Durdur | İptal Et |
|---|---|---|---|---|---|
| Finans / Uyum (raporlama) | ✓ | ✓ | ✓ | ✓ | ✓ |

## 4. Menü ve Navigasyon

Operasyonel Süreç Yönetimi menüsü altında.

## 5. Ekran Yapısı ve Bileşenler

Tek tablo. **Kolonlar:** Tarih, Rapor Adı, Status (`integration_status`).

Actions kolonunda Update/Delete yoktur; **Tekrar Dene / İşlemi Durdur / İptal Et** butonları (statüye göre). View ile ham JSON request/response detay panelinde gösterilir.

## 6. Alan Listesi

| Kolon | Tip | Açıklama |
|---|---|---|
| Tarih | date | Rapor dönemi/günü |
| Rapor Adı | string | BTRANS rapor tipi |
| Status | enum | integration_status |

## 7. Validasyon Kuralları

Salt-okunur izleme ekranı; aksiyonlar buton bazlıdır.

## 8. İş Kuralları ve Akışlar

- Statü akışı ve aksiyon kuralları 8.3 ile aynıdır (`0.10`).
- Error* → Tekrar Dene / İşlemi Durdur / İptal Et; OnHold → Tekrar Dene / İptal Et.
- Yeniden denemede dış sistemde kaydın varlığı kontrol edilir.

## 9. Statü Modeli

`integration_status` — bkz. `0.10`.

## 10. Veri Modeli

**`integration_record`** (BTRANS tipi): ortak entegrasyon tablosu; ek `report_name, report_date`.

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /integrations/btrans` | GET | Liste |
| `GET /integrations/btrans/{id}` | GET | Detay + ham JSON |
| `POST /integrations/btrans/{id}/retry` | POST | Tekrar dene |
| `POST /integrations/btrans/{id}/hold` | POST | İşlemi durdur |
| `POST /integrations/btrans/{id}/cancel` | POST | İptal et |

## 12. Entegrasyonlar

BTRANS (dış raporlama sistemi).

## 13. Yetkilendirme ve Güvenlik

- Finans / Uyum raporlama rolleri.

## 14. Loglama ve Denetim İzi

- Aksiyonlar correlation_id ile loglanır.

## 15. Hata Yönetimi

- Hata kayıtları statü ile gösterilir; ham JSON detayda.

## 16. Bildirimler

- Kritik rapor gönderim hatalarında uyarı (opsiyonel).

## 17. Batch / Zamanlanmış İşler

- BTRANS raporlarının periyodik üretilmesi ve gönderilmesi.

## 18. Bağımlılıklar

Para Transferleri (5), Raporlar (11), Entegrasyonlar (12.7).

## 19. Açık Sorular ve Riskler

- BTRANS rapor tipleri ve periyotları (günlük/aylık) Veri Sözlüğü/parametre ile netleştirilmeli.
