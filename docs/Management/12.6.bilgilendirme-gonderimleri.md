# 12.6 Bilgilendirme Gönderimleri

> Ortak kurallar için bkz. `0.genel-standartlar.md` (özellikle `0.16`).

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Sistem Yönetimi › Bilgilendirme Gönderimleri |
| Dosya | `12.6.bilgilendirme-gonderimleri.md` |
| Ekran tipi | Listeleme + detay (tablı) |
| İlgili ENUM | notification_type, notification_status |

## 2. Amaç ve Kapsam

SMS, E-posta ve Push bilgilendirme şablonlarının oluşturulduğu, yönetildiği ve izlendiği ekrandır.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | Insert | Update | Manuel Tetikleme |
|---|---|---|---|---|
| Sistem Yöneticisi | ✓ | ✓ | ✓ | ✓ |

## 4. Menü ve Navigasyon

Sistem Yönetimi menüsü altında. Detay ekranı Tanımlama, Manüel Tetikleme, Log tablarından oluşur.

## 5. Ekran Yapısı ve Bileşenler

**Liste kolonları:** Şablon Adı, Bildirim Türü (`notification_type`), Konu, Son Tetiklenme Tarihi.

**Tablar:**
- **Tanımlama** — Şablon Adı, Bildirim Türü, Konu, Şablon İçeriği (parametre alanları: `{kullanici_adi}`, `{islem_no}` vb.), Açıklama.
- **Manüel Tetikleme** — Şablon (combobox), Alıcı Adresi, Parametreler (textarea), Zamanlama, Gerekçe.
- **Log** — Tarih, Şablon Adı, Bildirim Türü, Status (`notification_status`), Alıcı, Alıcı Adresi, Mesaj (salt-okunur).

## 6. Alan Listesi

| Alan | Tip | Açıklama |
|---|---|---|
| Şablon Adı / Konu / Açıklama | string | — |
| Bildirim Türü | enum | SMS / Email / Push |
| Şablon İçeriği | text | Parametrik placeholder destekler |
| Alıcı Adresi / Parametreler / Zamanlama / Gerekçe | — | Manuel tetikleme |
| Status | enum | notification_status |

## 7. Validasyon Kuralları

- Şablon İçeriğindeki parametre placeholder'ları geçerli olmalı.
- Manuel tetiklemede Alıcı Adresi format kontrolü; Gerekçe zorunlu.

## 8. İş Kuralları ve Akışlar

- Şablon içeriğine değişken alanlar parametre olarak eklenebilir.
- Tekrar gönderim rate-limit ile sınırlıdır (bkz. `0.16`).
- Log tabı sistem tarafından doldurulur, değiştirilemez.

## 9. Statü Modeli

`notification_status` (gönderim sonucu).

## 10. Veri Modeli

**`notification_template`**: `id, name, notification_type, subject, content, description, last_triggered_at`
**`notification_log`**: `id, date, template_id, notification_type, status, recipient, recipient_address, message`

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /notification-templates` | GET | Liste |
| `POST /notification-templates` | POST | Yeni şablon |
| `PUT /notification-templates/{id}` | PUT | Güncelle |
| `POST /notification-templates/{id}/trigger` | POST | Manuel tetikle |
| `GET /notification-logs` | GET | Log |

## 12. Entegrasyonlar

SMS sağlayıcısı, e-posta (SMTP) sağlayıcısı, Push servisi.

## 13. Yetkilendirme ve Güvenlik

- Yalnızca Sistem Yöneticisi.
- Mesaj içeriği PII içerebilir; log ekranında maskeli özet, tam içerik yetkili erişimle.

## 14. Loglama ve Denetim İzi

- Şablon değişiklikleri ve manuel tetiklemeler (gerekçe) loglanır.

## 15. Hata Yönetimi

- Gönderim hatası Log'ta `notification_status` ile gösterilir.

## 16. Bildirimler

Bu ekran bildirim altyapısının kendisidir.

## 17. Batch / Zamanlanmış İşler

- Zamanlanmış bildirim gönderimleri (Manüel Tetikleme'deki Zamanlama).

## 18. Bağımlılıklar

Tüm bildirim tetikleyen modüller.

## 19. Açık Sorular ve Riskler

- Arapça şablonlar için RTL (sağdan sola) içerik desteği netleştirilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Şablon parametreleri gönderimde doğru değerlerle dolar.
- [ ] Tekrar gönderim rate-limit'e tabidir.
- [ ] Log tabı gönderim sonuçlarını gösterir.
- [ ] Mesaj içeriği log ekranında maskeli sunulur.
