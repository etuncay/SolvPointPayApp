# 7.5.1 Dolandırıcılık Bildir

> Ortak kurallar için bkz. `0.genel-standartlar.md`. Üst ekran: `7.5.vaka-inceleme.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Risk ve Uyum › Vaka İnceleme › Dolandırıcılık Bildir |
| Dosya | `7.5.1.dolandiricilik-bildir.md` |
| Ekran tipi | Modal/form |
| İlgili ENUM | fraud_type, fraud_detection_source, fraud_verdict |

## 2. Amaç ve Kapsam

Bir işlem için manuel **dolandırıcılık kaydının** girildiği ve gerekirse vaka açıldığı modal/formdur.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | Kaydet | Kaydet ve Vaka Aç |
|---|---|---|
| Uyum Operatör / Yönetici | ✓ | ✓ |

## 4. Menü ve Navigasyon

`7.5` Vaka İnceleme ekranındaki "Dolandırıcılık Bildir" butonu ile modal açılır.

## 5. Ekran Yapısı ve Bileşenler

**Butonlar:** Kaydet, Kaydet ve Vaka Aç, İptal.

**Alanlar:** İşlem Numarası, Dolandırıcılık Tipi (`fraud_type`), Tespit Kaynağı (`fraud_detection_source`), Hüküm (`fraud_verdict` — default Unknown), Tespit Tarih/Saati, Zarar Tutarı, Kurtarılan Tutar, Notlar (textarea).

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| İşlem Numarası | string | ✓ | — |
| Dolandırıcılık Tipi | enum | koşullu | Hüküm=ConfirmedFraud ise zorunlu |
| Tespit Kaynağı | enum | ✓ | — |
| Hüküm | enum | ✓ | Default Unknown |
| Tespit Tarih/Saati | datetime | ✓ | İşlem tarihinden önce olamaz |
| Zarar Tutarı / Kurtarılan Tutar | number | koşullu | Hüküm=NotFraud ise 0 olmalı |
| Notlar | text | — | Regex pattern kontrolü |

## 7. Validasyon Kuralları

- Hüküm = NotFraud ise Zarar Tutarı ve Kurtarılan Tutar **0** olmalı.
- Hüküm = ConfirmedFraud ise Dolandırıcılık Tipi **zorunlu**.
- Tespit Tarihi işlem tarihinden önce olamaz.
- Aynı İşlem Numarası için tekrar kayıt girilemez; mevcut kayıt varsa bilgileri getirilir, kullanıcı değiştirebilir.

## 8. İş Kuralları ve Akışlar

- İşlem ile ilişkili vaka varsa `linked_case_id` otomatik dolar.
- İşlemle ilişkili vaka varken kullanıcı "Kaydet ve Vaka Aç" tıklarsa **uyarı** verilir.
- "Kaydet" yalnızca dolandırıcılık kaydı oluşturur; "Kaydet ve Vaka Aç" ek olarak vaka açar.

## 9. Statü Modeli

`fraud_verdict`: Unknown / NotFraud / ConfirmedFraud / PreventedFraud.

## 10. Veri Modeli

**`fraud_record`**: `transaction_id, type (fraud_type), source (fraud_detection_source), verdict (fraud_verdict), fraud_discovery_at, fraud_labeler_user_id, loss_amount, recovered_amount, linked_case_id (nullable), notes`

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /fraud-records/{transactionId}` | GET | Mevcut kayıt (varsa) |
| `POST /fraud-records` | POST | Kaydet |
| `POST /fraud-records/with-case` | POST | Kaydet ve vaka aç |

## 12. Entegrasyonlar

Faz-2: SAR/STR otomatik bildirimi (uyum görevlisi onayı sonrası).

## 13. Yetkilendirme ve Güvenlik

- Uyum rolleri.

## 14. Loglama ve Denetim İzi

- Dolandırıcılık kaydı oluşturma/güncelleme loglanır.

## 15. Hata Yönetimi

- Validasyon ihlalleri (Hüküm-tutar/tip tutarsızlığı, geçmiş tarih) anlamlı hata.

## 16. Bildirimler

- Vaka açıldığında ilgili gruba/sorumluya bildirim.

## 17. Batch / Zamanlanmış İşler

Yoktur.

## 18. Bağımlılıklar

Vaka İnceleme (7.5), Para Transferleri (5), MASAK raporları (11).

## 19. Açık Sorular ve Riskler

- `fraud_type`, `fraud_detection_source`: öneri değer kümeleri bkz. [`Veri-Sozlugu-ENUM.md`](../Veri-Sozlugu-ENUM.md) §7.8–7.9; iş birimi teyidi bekleniyor.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Hüküm=NotFraud iken zarar/kurtarılan tutar 0 zorunludur.
- [ ] Hüküm=ConfirmedFraud iken Dolandırıcılık Tipi zorunludur.
- [ ] Tespit Tarihi işlem tarihinden önce olamaz.
- [ ] Aynı işlem için tekrar kayıt engellenir, mevcut kayıt getirilir.
- [ ] İlişkili vaka varken "Kaydet ve Vaka Aç" uyarı verir.
