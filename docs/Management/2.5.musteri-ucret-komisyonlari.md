## 2. Amaç ve Kapsam

Müşterilere uygulanacak **standart ücret ve komisyon tarifesinin** yönetildiği ekrandır. Müşteri tarafında tek bir ücret yapısı esastır; farklılaşma kampanyalar (2.6) ile yönetilir.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | Insert | Update | Export |
|---|---|---|---|---|
| Finans / Fiyatlandırma | ✓ | ✓ | ✓ | ✓ |
| Operasyon | ✓ | — | — | ✓ |

Ücret değişiklikleri Onay Havuzu'na (8.1) tabi olabilir (Onay Kuralları 12.2'ye göre).

## 4. Menü ve Navigasyon

Müşteriler menüsü altında; satır içi ekleme/güncelleme.

## 5. Ekran Yapısı ve Bileşenler

Tek tablo. **Kolonlar:** İşlem (`transaction_type`), Para Birimi (`currency`), Alt Limit, Sabit Ücret, Değişken Ücret (%), Başlangıç Tarihi, Bitiş Tarihi, Kaynak Ülke, Hedef Ülke, Değiştiren Kullanıcı, Değişiklik Tarihi, Status (`status`). Varsayılan sıralama: aktif kayıtlar üstte.

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| İşlem | enum | ✓ | transaction_type |
| Para Birimi | enum | ✓ | currency |
| Alt Limit | number | ✓ | Skalanın alt eşiği; ≥ 0 |
| Sabit Ücret | number | ✓ | — |
| Değişken Ücret (%) | number | ✓ | Oransal komisyon |
| Başlangıç/Bitiş Tarihi | date | — | Opsiyonel; boş başlangıç = giriş tarihi |
| Kaynak / Hedef Ülke | enum | — | "Tüm ülkeler" desteklenir |
| Değiştiren Kullanıcı / Değişiklik Tarihi | — | sistem | Otomatik |
| Status | enum | sistem | Active/Passive |

## 7. Validasyon Kuralları

- Alt Limit ≥ 0; Sabit Ücret ve Değişken Ücret negatif olamaz.
- Her İşlem + Para Birimi için en az bir "tüm ülkeler – tüm ülkeler" ve **Alt Limit = 0** aktif kayıt bulunmalı; aksi halde ücret hesaplanamaz.

## 8. İş Kuralları ve Akışlar

- Aynı İşlem + Para Birimi + Alt Limit + ülke kombinasyonu için yeni kayıt girilince mevcut kayıt pasife alınır (Bitiş Tarihi + Status otomatik).
- Tabloda olmayan yeni bir Alt Limit değeri **yeni kademe** olarak eklenir; mevcut kademeler korunur.
- **Kademe hesaplama:** işlem tutarına eşit veya küçük en yüksek Alt Limit'in tanımı uygulanır. Örn. 0→%5, 10.000→%2, 20.000→%1 ⇒ 15.000 TL işlem %2.
- **Ülke önceliği:** (Kaynak+Hedef spesifik) > (Kaynak spesifik + hedef tüm) > (Kaynak tüm + hedef spesifik) > (tüm+tüm). Eşitlikte en yüksek Alt Limit.
- Ücret hesabı işlem tarihinde geçerli aktif kayıtlar üzerinden yapılır.
- Başlangıcı gelmemiş veya Bitişi geçmiş kayıtlar günlük batch ile pasife alınır.

## 9. Statü Modeli

`status`: Active / Passive. Tarihçeli (her değişiklik yeni satır).

## 10. Veri Modeli

**`customer_fee`**: `id, transaction_type, currency, lower_limit, fixed_fee, variable_fee_pct, start_date, end_date, source_country, target_country, status, changed_by, changed_at, record_status`

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /customer-fees` | GET | Liste |
| `POST /customer-fees` | POST | Ekle (yeni kademe / pasifleme) |
| `PUT /customer-fees/{id}` | PUT | Güncelle |
| `GET /customer-fees/calculate` | GET | Verilen işlem/tutar/ülke için ücret hesapla |

## 12. Entegrasyonlar

FX kuru (YP işlemlerde TL karşılığı için Kur/FX Yönetimi 8.6).

## 13. Yetkilendirme ve Güvenlik

- Fiyatlandırma rolü; değişiklik Onay Havuzu'na tabi olabilir.

## 14. Loglama ve Denetim İzi

- Ücret ekleme/güncelleme eski-yeni değer ile loglanır.

## 15. Hata Yönetimi

- "Tüm ülkeler – Alt Limit 0" kaydı yoksa hesaplama servisi anlamlı hata kodu döner.

## 16. Bildirimler

Tetiklenmez.

## 17. Batch / Zamanlanmış İşler

- Günlük: Başlangıcı gelmemiş / Bitişi geçmiş ücret kayıtlarının pasife alınması.

## 18. Bağımlılıklar

Para Transferleri (ücret tüketicisi), Kampanya Yönetimi (2.6), Kur/FX (8.6).

## 19. Açık Sorular ve Riskler

- Ücret değişikliğinin Onay Havuzu'na tabi olup olmayacağı Onay Kuralları (12.2) ile netleştirilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Alt Limit = 0 / tüm ülkeler kaydı olmadan ücret hesaplanamaz.
- [ ] Kademe seçimi "≤ tutar en yüksek Alt Limit" mantığıyla çalışır.
- [ ] Ülke önceliği doğru uygulanır.
- [ ] Yeni kayıt mevcut çakışan kaydı pasife alır.
- [ ] Batch ile süresi dolan kayıtlar pasifleşir.
