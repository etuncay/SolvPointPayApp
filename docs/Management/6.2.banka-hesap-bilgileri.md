# 6.2 Banka Hesap Bilgileri

> Ortak kurallar için bkz. `0.genel-standartlar.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Bankalar › Banka Hesap Bilgileri |
| Dosya | `6.2.banka-hesap-bilgileri.md` |
| Ekran tipi | Listeleme + satır içi ekleme/güncelleme |
| İlgili ENUM | bank_account_type, entity_status |

## 2. Amaç ve Kapsam

Firmaya ait banka hesaplarının (cari, koruma vb.) tanımlandığı ekrandır. Bakiye bilgisi dış entegrasyonla otomatik doldurulur.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | Insert | Update | Delete |
|---|---|---|---|---|
| Finans / Muhasebe | ✓ | ✓ | ✓ | (pasife alma) |

## 4. Menü ve Navigasyon

Bankalar menüsü altında; satır içi ekleme/güncelleme.

## 5. Ekran Yapısı ve Bileşenler

Tek tablo. **Kolonlar:** Banka, Hesap Tipi (`bank_account_type`), IBAN, Para Birimi, Bakiye, Şube Kodu, Hesap No, Ek No, Son Güncelleme Zamanı, Status (`entity_status`).

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| Banka | enum | ✓ | Entegre banka listesi |
| Hesap Tipi | enum | ✓ | bank_account_type (cari, koruma vb.) |
| IBAN | string | ✓ | Sistemde tek aktif kayıt |
| Para Birimi | enum | ✓ | — |
| Bakiye | number | sistem | Dış entegrasyonla otomatik; değiştirilemez |
| Şube Kodu / Hesap No / Ek No | string | ✓/✓/— | — |
| Son Güncelleme Zamanı | datetime | sistem | — |
| Status | enum | sistem | entity_status |

## 7. Validasyon Kuralları

- IBAN formatı + checksum kontrolü.
- Aynı IBAN sistemde **birden fazla kez aktif** tanımlanamaz.

## 8. İş Kuralları ve Akışlar

- Bakiye kullanıcı tarafından girilmez/değiştirilemez; banka entegrasyonu ile otomatik doldurulur.

## 9. Statü Modeli

`entity_status`: Active / Inactive.

## 10. Veri Modeli

**`company_bank_account`**: `id, bank_id, account_type, iban, currency, balance, branch_code, account_no, suffix, last_updated_at, status, created_at/by, updated_at/by, record_status`

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /company-bank-accounts` | GET | Liste |
| `POST /company-bank-accounts` | POST | Ekle |
| `PUT /company-bank-accounts/{id}` | PUT | Güncelle |
| `GET /company-bank-accounts/{id}/balance` | GET | Bakiye sorgu (entegrasyon) |

## 12. Entegrasyonlar

Banka bakiye sorgu entegrasyonu (otomatik bakiye güncelleme).

## 13. Yetkilendirme ve Güvenlik

- Finans / Muhasebe rolleri.

## 14. Loglama ve Denetim İzi

- Hesap ekleme/güncelleme loglanır; bakiye otomatik güncellemeleri loglanır.

## 15. Hata Yönetimi

- Mükerrer aktif IBAN girişinde anlamlı hata.

## 16. Bildirimler

Tetiklenmez.

## 17. Batch / Zamanlanmış İşler

- Periyodik banka bakiye senkronizasyonu (entegrasyon).

## 18. Bağımlılıklar

Entegre Bankalar (6.1), Banka Hesap Hareketleri (6.3), Finansal Mutabakat (8.5).

## 19. Açık Sorular ve Riskler

- Koruma hesabının (banka tarafından bloke, gün içi tek işlem) bu ekrandaki `bank_account_type` ile nasıl ayrıştırılacağı netleştirilmeli.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Aynı IBAN birden fazla kez aktif tanımlanamaz.
- [ ] Bakiye kullanıcıdan girilmez, entegrasyonla dolar.
- [ ] IBAN checksum kontrolü çalışır.
