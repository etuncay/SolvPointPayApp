# 2.4 Müşteri Belge İnceleme

> Ortak kurallar için bkz. `0.genel-standartlar.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Müşteriler › Müşteri Belge İnceleme |
| Dosya | `2.4.musteri-belge-inceleme.md` |
| Ekran tipi | Listeleme (grid) — kuyruk |
| Detay ekranı | `2.4.1.musteri-belge-inceleme-detay.md` |
| İlgili ENUM | document_category, approval_status, identity_document, customer_type |

## 2. Amaç ve Kapsam

Kontrol edilmemiş müşteri dokümanlarının (`approval_required=true && approval_status=Pending`) listelendiği inceleme kuyruğu ekranıdır. Uyum/operasyon personeli buradan belge detayına geçer ve karar verir.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | View (Detay) | Karar |
|---|---|---|---|
| Uyum | ✓ | ✓ | ✓ |
| Operasyon (belge onay yetkili) | ✓ | ✓ | ✓ |

Belge türü bazlı erişim: Genel müdürlük personeli yalnızca yetkili olduğu doküman türlerini görür (örn. "Kimlik Belgeleri" yalnızca Uyum).

## 4. Menü ve Navigasyon

Satıra tıklandığında `2.4.1 Müşteri Belge İnceleme Detay` açılır.

## 5. Ekran Yapısı ve Bileşenler

Tek tablo. **Kolonlar:** Müşteri No, Adı ve Soyadı / Unvanı, Uyruğu, Şüpheli İşareti, Oluşturma Tarihi, Belge Kategorisi (`document_category`), Belge Türü, Belge İncelenme Durumu (`approval_status`).

## 6. Alan Listesi

| Kolon | Tip | Açıklama |
|---|---|---|
| Müşteri No | string | — |
| Adı ve Soyadı / Unvanı | string | — |
| Uyruğu | enum | Ülke |
| Şüpheli İşareti | bool | Temsilci tarafından işaretlenmiş şüpheli kayıt |
| Oluşturma Tarihi | datetime | — |
| Belge Kategorisi / Türü | enum / string | — |
| Belge İncelenme Durumu | enum | approval_status (Pending) |

## 7. Validasyon Kuralları

Listeleme ekranıdır; form validasyonu yoktur.

## 8. İş Kuralları ve Akışlar

- Yalnızca `approval_required=true && approval_status=Pending` belgeler listelenir. Taslak müşteri belgeleri de listede yer alır.
- **Varsayılan sıralama:** (1) Şüpheli=True ve Uyruğu≠TUR, (2) Şüpheli=True, (3) Uyruğu≠TUR, (4) diğerleri — her grup kendi içinde Oluşturma Tarihi eskiden yeniye.

## 9. Statü Modeli

`approval_status`: Pending (kuyrukta) → Approved / Rejected (detayda karar verilince). Belge `document_status` ile birlikte yönetilir (bkz. `0.12`).

## 10. Veri Modeli

Kaynak `dms.document` tablosu (bkz. `9.dokuman-yonetim-sistemi.md`). Bu ekran filtreli okuma yapar; `customer` ile ilişkilidir.

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /customer-documents/review-queue` | GET | İnceleme kuyruğu (sıralı) |
