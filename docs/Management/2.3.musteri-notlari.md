# 2.3 Müşteri Notları

> Ortak kurallar için bkz. `0.genel-standartlar.md`.

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Müşteriler › Müşteri Notları |
| Dosya | `2.3.musteri-notlari.md` |
| Ekran tipi | Listeleme + satır içi ekleme/güncelleme (detay sayfası yok) |
| İlgili ENUM | entity_type_core, level |

## 2. Amaç ve Kapsam

Temsilcinin müşteri ekranını veya müşterinin kendi ekranını açtığında **gösterilmesi istenen bilgilendirme notlarının** tanımlandığı ekrandır.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | Insert | Update | Delete |
|---|---|---|---|---|
| Operasyon | ✓ | ✓ | ✓ | ✓ |
| Uyum | ✓ | ✓ | ✓ | ✓ |

## 4. Menü ve Navigasyon

Müşteriler menüsü altında; satır içi ekleme/güncelleme. Detay sayfası bulunmaz.

## 5. Ekran Yapısı ve Bileşenler

Tek tablo. Yeni eklemede en alta boş satır açılır; güncellemede satır değerleri düzenlenebilir hale gelir (alternatif: modal — ekip tercihine bağlı).

**Kolonlar:** Müşteri No, Bilgilendirme Metni, Bilgilendirilen (`entity_type_core`), Öncelik Seviyesi (`level`), Gösterim Limiti, Gösterim Adedi, Bitiş Tarihi.

## 6. Alan Listesi

| Alan | Tip | Zorunlu | Açıklama |
|---|---|---|---|
| Müşteri No | string | ✓ | Notun ait olduğu müşteri |
| Bilgilendirme Metni | string | ✓ | Gösterilecek metin |
| Bilgilendirilen | enum | ✓ | Notu kimin göreceği (temsilci/müşteri vb.) |
| Öncelik Seviyesi | enum | ✓ | level (Low/Medium/High/Critical) |
| Gösterim Limiti | int | — | Opsiyonel; not en fazla bu kadar gösterilir |
| Gösterim Adedi | int | sistem | Salt-okunur; her görüntülemede +1 |
| Bitiş Tarihi | date | — | Doluysa bu tarihten sonra gösterilmez |

## 7. Validasyon Kuralları

- Bilgilendirme Metni: boş olamaz; regex pattern kontrolü (TCKN/kart no engeli); max uzunluk.
- Gösterim Limiti: girilirse pozitif tam sayı.
- Bitiş Tarihi: girilirse geçerli tarih.

## 8. İş Kuralları ve Akışlar

- **Gösterim Adedi** sistemseldir; not her görüntülendiğinde otomatik artar, kullanıcı değiştiremez.
- **Gösterim Limiti** doluysa not ilgili kullanıcıya en fazla limit kadar gösterilir.
- **Bitiş Tarihi** doluysa o tarihten sonra not gösterilmez.
- Notlar Temsilci/Müşteri uygulamalarında sayfanın üstünde uyarı olarak listelenir.

## 9. Statü Modeli

Bu ekrana özgü statü modeli yoktur (record_status ile yaşam döngüsü).

## 10. Veri Modeli
