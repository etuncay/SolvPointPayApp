# 9. Doküman Yönetim Sistemi

> Ortak kurallar için bkz. `0.genel-standartlar.md` (özellikle `0.12 Doküman Yönetimi`).

## 1. Doküman Künyesi

| Alan | Değer |
|---|---|
| Menü | Doküman Yönetim Sistemi |
| Dosya | `9.dokuman-yonetim-sistemi.md` |
| Ekran tipi | Listeleme (grid) |
| Alt ekranlar | 9.1 Yeni Doküman Ekle, 9.2 Doküman Detay, 9.3 Doküman Tipleri, 9.4 Yeni Doküman Tipi Tanımlama |
| İlgili ENUM | document_category, document_status |

## 2. Amaç ve Kapsam

Sistemdeki tüm yüklü dokümanların (müşteri, temsilci, işlem, talep, çalışan belgeleri) merkezi olarak listelendiği ve yönetildiği ekrandır.

## 3. Aktörler ve Rol/Yetki Matrisi

| Aktör | List | View (Detay) | Insert |
|---|---|---|---|
| Yetkili personel | belge türü bazlı | belge türü bazlı | ✓ |

Belge türü bazlı erişim: Doküman Tipleri'nde (9.4) tanımlı "Görüntüleyebilen Roller". Approve yetkisi olanın otomatik view yetkisi de vardır.

## 4. Menü ve Navigasyon

Sol menüde "Doküman Yönetim Sistemi"; altında 9.1–9.4. Müşteri/Temsilci ekranlarındaki "Belge Görme/İndirme" bu ekranı ilgili ID filtreli açar.

## 5. Ekran Yapısı ve Bileşenler

Tek panelli grid. **Kolonlar:** Oluşturulma Tarihi, Doküman Kategorisi (`document_category`), Doküman Türü, Status (`document_status`), Oluşturan, Onaylayan.

## 6. Alan Listesi

| Kolon | Tip | Aranabilir | Açıklama |
|---|---|---|---|
| Oluşturulma Tarihi | datetime | ✓ (aralık) | — |
| Doküman Kategorisi / Türü | enum / string | ✓ | — |
| Status | enum | ✓ | document_status |
| Oluşturan / Onaylayan | user | ✓ | — |

## 7. Validasyon Kuralları

Listeleme ekranıdır; ekleme validasyonu 9.1'de.

## 8. İş Kuralları ve Akışlar

- Belge türü bazlı görünürlük backend'de enforce edilir.
- `Active` ve `Archived` dokümanlar kullanılabilir; `Rejected/Expired/Inactive` dokümanlar kanıt amaçlı saklanır.
- Belge yaşam döngüsü için bkz. `0.12`.

## 9. Statü Modeli

`document_status`: Inactive / Active / Rejected / Expired / Archived. `approval_status`: NULL (onay gerekmez) / Pending / Approved / Rejected.

## 10. Veri Modeli

**`dms.document`**: `id, document_category, document_type_id, document_status, approval_required, approval_status, file_name, file_size, file_type, file_hash (SHA-256), storage_path, valid_from, valid_until, is_personal_data, active_retention_period, archive_retention_period, created_at/by, approved_at/by, record_status`

**`dms.document_relation`**: `id, document_id, relation_type (Customer/Agent/Transaction/Complaint/Employee), related_id`

## 11. Servis / API Sözleşmeleri

| Servis | Method | Açıklama |
|---|---|---|
| `GET /documents` | GET | Filtreli liste (rol/belge türü bazlı) |
| `GET /documents/{id}` | GET | Detay (9.2) |
| `GET /documents/{id}/download` | GET | İndir |

## 12. Entegrasyonlar

Antivirüs/malware tarama servisi; OCR (kimlik belgesi) servisi; depolama (storage).

## 13. Yetkilendirme ve Güvenlik

- Belge türü bazlı görüntüleme/onaylama yetkisi.
- Yükleme güvenliği: uzantı + content-type + gerçek mime; mimetype sniffing engeli; güvenli indirme header'ları.

## 14. Loglama ve Denetim İzi

- Belge erişim/indirme PII erişimi olarak loglanır.

## 15. Hata Yönetimi

- Tarama tamamlanana kadar belgeye statü atanmaz; tarama başarısızsa karantina.

## 16. Bildirimler

Tetiklenmez (onay bildirimleri ilgili inceleme ekranlarında).

## 17. Batch / Zamanlanmış İşler

- Günlük: `valid_until` takibi — süresi dolan dokümanların arşive kaldırılması (gerekirse KYC seviyesi/statü değişimi).
- Aylık: Integrity check — depolamadan okuma + yeniden hash + `file_hash` karşılaştırma; hatalılar uyum görevlisine mail.

## 18. Bağımlılıklar

Müşteriler (2), Temsilciler (3), Para Transferleri (5), Destek Merkezi (10), İnsan Kaynakları (13), Doküman Tipleri (9.3).

## 19. Açık Sorular ve Riskler

- Belge türü bazlı görünürlük matrisinin 9.4 "Görüntüleyebilen Roller" ile tek kaynaktan yönetildiğinden emin olunmalı.

## 20. Kabul Kriterleri ve Test Senaryoları

- [ ] Belge türü bazlı görünürlük çalışır.
- [ ] Active/Archived kullanılabilir, diğer statüler kanıt amaçlı saklanır.
- [ ] Süresi dolan belge batch ile arşive alınır.
- [ ] Aylık integrity check hatalı hash'leri tespit eder.
