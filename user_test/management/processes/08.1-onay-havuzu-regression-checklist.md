# Onay Havuzu — Approve/Reject Store + UI Regression Checklist

Approve veya reject sonrası mock store ile liste/detay/widget görünümünün tutarlı kaldığını doğrular. Otomatik store testleri: `approval-store-regression.test.ts`.

## Ön koşullar

- [ ] `pnpm --filter @epay/management dev` → mock mod (`DemoModeBanner` görünür)
- [ ] Rol: `compliance` (1. onay) veya `alltest` (tam erişim)
- [ ] Seed: `mocks/approval-requests.ts` — id `1` tek onaylı bekleyen

## Otomatik (Vitest)

```bash
pnpm --filter @epay/management exec vitest run \
  src/features/approval-pool/api/approval-store-regression.test.ts \
  src/features/approval-pool/api/mock-approvals-adapter.test.ts \
  src/features/approval-pool/domain/transitions.test.ts
```

| Senaryo | Beklenen |
|---------|----------|
| 1-onaylı approve | `uiStatus=approved`, `pending_mine` düşer, `approved_mine` artar |
| 1-onaylı reject | `uiStatus=rejected`, `rejected_mine` artar |
| 2-onaylı 1. approve | `awaiting_second`, 2. onaycı `pending_mine` |
| 2-onaylı 2. approve / 2. red | `approved` / `second_rejected` |
| Widget | `pendingApprovalWidgetRows` = `pending_mine` ilk 8 |
| Mükerrer approve | Store değişmez |
